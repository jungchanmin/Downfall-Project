# Downfall Project Manifest

> Bootstrap manifest. The detailed file inventory is generated by `tools/repository/build_manifest.py` and maintained by GitHub Actions.

## Repository identity

- Repository: `jungchanmin/Downfall-Project`
- Default branch: `main`
- Primary documentation root: `Wiki/`
- Runtime code root: `Scripts/`
- Tooling root: `tools/`
- Automation root: `.github/workflows/`

## Generated inventory

The complete deterministic inventory is produced with:

```bash
python tools/repository/build_manifest.py
```

CI regenerates this file after repository-management, Wiki, code, or workflow changes. Do not maintain the generated inventory manually.

## Known management limitations

- File presence does not prove that a Unity asset or script is referenced at runtime.
- Scene, Prefab and `.meta` GUID integrity requires Unity-aware validation.
- Binary assets require separate review and may require Git LFS.
- Deletion candidates must be reference-searched and approved before removal.
