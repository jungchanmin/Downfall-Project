# 📢 통보형 이벤트 템플릿 (Track A: Notification Event)

## ⚙️ 1. 메타데이터 (Metadata)
*이벤트의 고유 식별자와 시공간적 제한을 정의합니다.*
```yaml
이벤트_ID: "EV_NOTI_[분류]_[순번]"  # 예: EV_NOTI_Conflict_001
이벤트_분류: ["세계", "NPC_단독", "NPC_상호작용", "시스템"]
발생_페이즈: ["새벽", "아침", "점심", "저녁", "밤"] # 복수 선택 가능
발생_장소: ["아지트", "Any", "특정_구역명"]
```

## 🏷️ 2. 태그형 발동 조건 (Trigger Tags)
*이 조건 태그를 만족하는 인물들이 자동으로 하단의 `[NPC_A]`, `[NPC_B]` 변수에 할당(Casting)됩니다.*
```yaml
주체_조건_태그(NPC_A): 
  - "#불만도_8이상"
  - "#스트레스_7이상"
대상_조건_태그(NPC_B): 
  - "#NPC_A와_동일구역"
  - "#NPC_A와의_친밀도_3이하"
환경_조건_태그: 
  - "#식량_0"
연계_트리거_태그: 
  - "#이전_이벤트(EV_NOTI_Starvation_001)_발생_후_1일_경과"
```

## 📖 3. 상황 내러티브 (Narrative Script)
*UI에 출력되는 텍스트입니다. 절대 특정 인물의 이름을 직접 쓰지 않고 변수(`[NPC_A]`, `[NPC_B]`)를 사용합니다.*

- **이벤트 요약**: 굶주림과 불만도로 인한 생존자 간의 폭력적 다툼.
- **출력 텍스트**:
  > "극심한 배고픔이 이성의 끈을 끊어놓았습니다. **[NPC_A]**가 구석에 앉아있던 **[NPC_B]**의 멱살을 잡고 배급량에 대해 거칠게 항의하기 시작합니다. **[NPC_B]** 역시 지지 않고 욕설을 내뱉으며 분위기가 험악해집니다. 이들의 고성이 거점을 울립니다."

## 📉 4. 자동 결과 및 수치 변동 (Auto-Results)
*플레이어의 개입(선택지) 없이, 엔진이 즉각적으로 스탯에 반영하고 팝업을 종료하는 수치들입니다.*

- **스탯 변동**:
  - `[NPC_A]` : 스트레스 -1 (분출함), 체력 -1 (난투극)
  - `[NPC_B]` : 스트레스 +2, 체력 -1 (난투극)
  - `[NPC_A] <-> [NPC_B]` : 상호 친밀도 -3
  - `[아지트_전체]` : 동일 구역 내 모든 생존자 스트레스 +1
- **연계 이벤트 발행(Publish)**: 
  - 이벤트 종료 시 큐(Queue)에 다음 태그 발행: `#EV_NOTI_Conflict_001_발생됨` (추후 파벌 갈등 이벤트의 트리거로 작동)
```
