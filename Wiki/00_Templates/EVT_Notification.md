# 📢 통보형 이벤트 템플릿 (Track A: Notification Event)

## ⚙️ 1. 메타데이터 (Metadata)
```yaml
이벤트_ID: "EV_NOTI_[분류]_[순번]"
이벤트_분류: ["세계", "NPC_단독", "NPC_상호작용", "시스템"]
발생_페이즈: ["새벽", "아침", "점심", "저녁", "밤"]
발생_장소: ["아지트", "Any", "특정_구역명"]
```

## 🏷️ 2. 태그 관리 및 명명 규칙 (Tag Schema)
- **#Stat_[스탯명]_[연산자][수치]**: (예: `#Stat_불만도_>=8`)
- **#Trait_[특성명]**: (예: `#Trait_ENTJ`, `#Trait_부상`)
- **#Loc_[장소명]**: (예: `#Loc_아지트_거실`, `#Loc_SameAs_NPC_A`)
- **#Env_[변수]_[연산자][수치]**: (예: `#Env_식량_==0`)
- **#Flag_[ID]_[상태]**: (예: `#Flag_EV_001_True`)

## 🎯 3. 발동 조건 (Trigger Conditions)
```yaml
주체_조건(NPC_A): ["#태그1", "#태그2"]
대상_조건(NPC_B): ["#태그1", "#태그2"]
환경_및_연계_조건: ["#태그1", "#태그2"]
```

## 🖋️ 4. 전역 문체 가이드: Downfall Voice
*AI는 모든 지문 작성 시 아래의 문학적 기준을 반드시 준수해야 합니다.*
- **건조한 하드보일드 문체**: 헤밍웨이 스타일을 지향함. 과장된 수식어, 형용사, 감탄사를 배제하고 짧고 간결한 문장으로 상황을 전달할 것.
- **감각적 묘사 (Show, Don't Tell)**: "절망적이다"라고 말하지 말고, "입가에 묻은 마른 침 자국"이나 "텅 빈 배급 상자 위를 기어가는 벌레"를 묘사하여 분위기를 조성할 것.
- **감정의 절제**: 슬픔이나 공포를 직접적으로 표현하는 대신, 인물의 떨리는 손끝이나 거친 호흡음으로 암시할 것.

## 🎭 5. 페르소나 은유 지침 (Subtle Persona Directive)
- **직업적 관점의 투영**: NPC의 직업(사업가, 의사 등)을 전문 용어 나열이 아닌, **'세상을 해석하는 방식'**으로 녹여낼 것. (예: 사업가는 모든 비극을 '손실'로, 의사는 '부패'로 바라봄)
- **절제된 대사**: 대사는 길지 않아야 함. 한두 마디의 무겁고 날카로운 문장이 인물의 성격을 대변해야 하며, 대화보다는 인물의 '시선'과 '태도' 묘사에 더 비중을 둘 것.
- **재사용성**: 텍스트 내에 실제 이름을 쓰지 말고 엔진 변수(`[NPC_A]`, `[NPC_B]`)를 유지할 것.

## 📖 6. 상황 내러티브 (Narrative Script)
- **이벤트 요약**: [이벤트의 핵심 내용]
- **출력 텍스트**:
  > "[NPC_A]의 시선이 [NPC_B]의 손에 쥔 빵조각에 머뭅니다. [NPC_A]의 눈에는 배고픔보다 더 깊은 무언가가 소용돌이치고 있습니다. '...그게 우리의 마지막 자원입니까?' 목소리는 낮고 서늘합니다."

## 📉 7. 자동 결과 및 수치 변동 (Auto-Results)
- **스탯 변동**: (예: `[NPC_A]` : `#Stat_불만도_+1`)
- **플래그 발행**: (예: `#Flag_[ID]_Done`)
```
