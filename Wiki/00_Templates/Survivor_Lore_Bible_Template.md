    *   [과거 설정을 언제, 누구에게, 어느 정도까지 말할 수 있는지.]
    *   [일반 이벤트에서 직접 설명하지 말아야 할 설정.]

*   **EVENT GENERATION RULE / 이벤트 생성 규칙**
    *   로어바이블은 장면에 모두 표현해야 하는 체크리스트가 아니다.
    *   장면의 목적과 현재 상황에 직접 관련된 설정만 사용한다.

    *   캐릭터의 성격은 해설보다 선택, 행동, 침묵, 거리감으로 표현한다.
    *   일반 이벤트에서는 `Behavioral Baseline`을 극단 상태보다 우선한다.
    *   이벤트 생성에는 이 문서 전문이 아니라 Runtime Character Card를 사용한다.

## 🔒 18. Canon & Data Boundaries — 설정과 데이터 경계

*   **LORE SOURCE OF TRUTH / 로어 원본**: 이 문서는 캐릭터의 고정 설정과 서사적 정체성의 기준 문서다.
*   **RUNTIME DERIVATIVE / 런타임 파생 문서**: `RUNTIME_CHAR_[영문 이름]`
*   **GAMEPLAY DATA SOURCE / 게임 데이터 원본**: `DATA_CHAR_[영문 이름]`
*   **NUMERIC DATA RULE / 수치 데이터 규칙**: 정확한 스탯, 스킬 계수, AI 가중치는 이 문서에 기록하지 않는다.
*   **CONFLICT PRIORITY / 충돌 우선순위**
    1. 현재 대화에서 승인된 최신 설정
    2. 본 로어바이블
    3. Runtime Character Card
    4. Gameplay Profile의 서술 정보
    5. 일반 생성 결과
*   **UPDATE RULE / 갱신 규칙**: 로어 변경 시 Runtime Card와 Gameplay Profile의 관련 항목을 함께 검토한다.

## ✅ 19. Authoring Checklist — 작성 및 검수 체크리스트

### 설정 보존

- [ ] 기존 문서의 설정이 누락되지 않았는가?
- [ ] 신규 내용에 `[신규 제안]`이 표시되었는가?
- [ ] 다른 캐릭터 및 세계관 문서와 충돌하지 않는가?
- [ ] 캐릭터의 직업·문화·과거가 행동에 자연스럽게 연결되는가?

### 행동 설계

- [ ] 평상시 행동과 극단 행동이 구분되어 있는가?
- [ ] 트리거가 없는데 폭주·고백·환각이 발생하지 않는가?
- [ ] 행동 빈도가 정의되어 있는가?
- [ ] 캐릭터가 하지 않는 행동이 명시되어 있는가?
- [ ] 관계 변화에 필요한 조건이 정의되어 있는가?

### 생성 안정성

- [ ] 특정 단어·상징·트라우마가 여러 항목에서 과도하게 반복되지 않는가?
- [ ] 예시 대사가 실제 생성 문장처럼 과도하게 구체적이지 않은가?
- [ ] 대표 대사를 재사용하지 말라는 규칙이 포함되어 있는가?
- [ ] 일반 이벤트에서 활용할 수 있는 평범한 행동이 충분한가?
- [ ] 설정 해설보다 관찰 가능한 행동 중심으로 작성되었는가?

### 데이터 분리

- [ ] 정확한 스탯 수치가 Gameplay Profile로 분리되었는가?
- [ ] 이벤트 생성 규칙이 Runtime Card로 파생 가능하게 작성되었는가?
- [ ] 세 문서가 동일한 `character_id`를 사용하는가?
- [ ] 로어 변경이 다른 문서에 미치는 영향이 확인되었는가?
