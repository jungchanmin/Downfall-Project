# 아지트 자율행동 및 명령 수행 프로필

## 1. 목적

이 문서는 아지트에 머무는 생존자가 아침·저녁에 수행하는 행동의 선택 구조와 플레이어 명령, 자율행동, 명령 저항, 강제 수행의 공통 규칙을 정의한다.

세부 수치, 임계값, 판정 난이도와 관계 변화량은 후속 밸런스 단계에서 확정한다.

## 2. 확정된 구조 결정

### 2.1 방 주 개조는 회차 영구 선택

- 각 방은 하나의 주 개조만 선택한다.
- 주 개조가 완성되면 같은 회차에서 다른 분기로 변경할 수 없다.
- 시설 파괴 후에는 선택했던 동일 분기를 복구할 수 있다.
- 파괴를 이용해 다른 개조로 전환할 수 없다.
- 회차 종료 후 다음 플레이에서는 다른 빌드를 선택할 수 있다.

이 규칙은 재개조 관리 부담을 줄이고, 다회차 플레이에서 아지트 빌드 지식을 보상한다.

### 2.2 설비 처리

- 설비는 주 개조 분기를 변경하지 않는다.
- 설치·교체 가능 여부는 설비별로 정의한다.
- 전용 설비가 파괴돼도 주 개조 선택은 유지된다.
- 개조와 무관한 휴대 장비는 일반 Item System을 따른다.

## 3. 행동 배정 방식

아지트 행동은 **플레이어 지정 + 미지정 생존자 자율 선택** 혼합형을 사용한다.

```text
행동 페이즈 시작
→ 행동 불능 여부 확인
→ 플레이어 명령 배정
→ 명령 수용 여부 평가
→ 미지정 생존자 자율행동 계산
→ 방 수용 인원과 설비 조건 확인
→ 행동 판정 및 결과 처리
→ 관계·스트레스·시설 상태 반영
```

### 3.1 플레이어 지정

플레이어는 우선순위가 높은 생존자에게 직접 행동과 방을 지정할 수 있다.

예:

- 헤이즐에게 작은방 의무실에서 전문 치료 지시
- 레이첼에게 정원 훈련 마당에서 전투 훈련 지시
- 가빈에게 거실 작전실에서 외출 정보 분석 지시

### 3.2 미지정 생존자

명령받지 않은 생존자는 자신의 상태, 성격, 관계와 현재 아지트 상황을 바탕으로 행동을 고른다.

플레이어는 세부 행동 대신 다음과 같은 우선 방침을 줄 수 있다.

- 자율
- 휴식 우선
- 치료 우선
- 관계 회복 우선
- 생산 우선
- 경계 우선

우선 방침은 강제 명령이 아니며 NPC 판단에 가중치만 준다.

## 4. 자율행동 평가

자율행동은 다음 요소를 평가한다.

| 요소 | 영향 예시 |
|---|---|
| 신체 상태 | 부상·피로가 높으면 휴식·치료 선호 |
| 스트레스 | 휴식·취미·대화 선호, 고위험 행동 회피 |
| 불만도 | 공동 작업·리더 지시 회피 가능 |
| 충성도 | 집단에 필요한 행동을 자발적으로 선택 |
| 친밀도 | 특정 인물과 같은 방·대화·간호 선호 |
| 성격·에고 | 공격성, 인내심, 양심, 외향성 등에 따른 선호 |
| 능력 적합성 | 잘하는 행동을 선택할 가능성 증가 |
| 기벽 | 특정 행동 강제 선호·회피·실패 위험 |
| 방 상태 | 손상·오염·점거된 방 회피 |
| 집단 필요 | 중상자, 파손 장비, 경계 공백 등 긴급도 |

### 4.1 에고

`에고`는 별도 수치로 반드시 추가하는 것이 아니라, 다음 요소를 합친 행동 성향 개념으로 우선 사용한다.

- 자기 판단을 고집하는 성향
- 명령과 개인 가치관의 충돌
- 자존심·직업 정체성
- 리더에 대한 신뢰
- 현재 불만과 감정

후속 NPC 행동 시스템에서 독립 파라미터가 필요하다고 판단될 때만 수치화한다.

## 5. 명령 수용 평가

모든 명령에 판정을 요구하지 않는다.

평상시의 합리적이고 적합한 명령은 자동 수용한다. 저항 평가는 다음 조건 중 하나 이상이 강할 때만 발생한다.

- 불만도가 높음
- 충성도가 낮음
- 스트레스·피로·부상이 심함
- 명령이 성격·가치관·기벽과 충돌
- 능력에 비해 지나치게 어려운 행동
- 위험한 방 또는 손상 시설 사용
- 친밀한 인물을 해치는 결과
- 반복적으로 같은 생존자에게 불리한 명령
- 리더의 직전 약속·결정을 뒤집는 명령

### 5.1 평가 요소

```text
명령 수용 경향
= 충성도
+ 리더 친밀도·신뢰
+ 명령 필요성
+ 능력 적합성
- 불만도
- 스트레스·피로
- 성격·기벽 충돌
- 개인 위험과 부당함
```

이 수식은 개념식이며 실제 수치는 후속 NPC 행동 계산기가 소유한다.

## 6. 명령 반응 단계

| 반응 | 행동 |
|---|---|
| 적극 협조 | 명령 수행, 상황에 따라 추가 이점 |
| 협조 | 정상 수행 |
| 마지못한 수용 | 수행하지만 스트레스·불만 또는 효율 페널티 가능 |
| 이의 제기 | 대화 선택지 발생, 즉시 수행하지 않음 |
| 거부 | 다른 행동을 선택하려 함 |
| 적극 저항 | 명령 거부와 갈등 이벤트 발생 가능 |

명령 저항은 단순 알림창이 아니라 캐릭터의 이유와 대사를 동반해야 한다.

## 7. 저항 발생 시 플레이어 선택

### 7.1 설득

- 대화 또는 관련 관계 판정
- 성공하면 자발적으로 수행
- 실패하면 행동 기회를 잃거나 거부 강화 가능
- 설득 방식은 NPC 성격에 따라 달라질 수 있음

### 7.2 양보·타협

- 더 안전하거나 적합한 행동으로 변경
- 보상·기호품·휴식·후속 약속 제시 가능
- 즉시 갈등은 줄지만 자원이나 다른 기회를 소비

### 7.3 인원 교체

- 다른 생존자에게 행동 배정
- 최초 거부자의 불만을 크게 올리지 않음
- 대체 인력의 능력과 피로를 감수

### 7.4 강제

- 명령을 강제로 수행시킴
- 해당 페이즈의 행동 결과는 얻을 수 있음
- 스트레스·불만 증가
- 충성도 또는 리더 신뢰 감소 가능
- 명령 성격에 따라 친밀도와 다른 생존자의 반응 변화
- 반복 강제 시 집단 갈등·반항·이탈 사건 가중

강제는 무료 확정 버튼이 아니다. 플레이어는 즉시 효율과 장기 관계 비용을 교환한다.

## 8. 강제 명령의 차등 대가

모든 강제가 같은 관계 손실을 주지는 않는다.

| 상황 | 대가 경향 |
|---|---|
| 명백한 생존 위기 대응 | 비교적 낮음 |
| 능력에 맞지만 피곤한 작업 | 낮음~중간 |
| 부상자에게 무리한 작업 | 높음 |
| 가치관·트라우마와 충돌 | 높음 |
| 특정 인물 희생·가혹 행위 | 매우 높음 |
| 반복적 편파 명령 | 누적 상승 |

다른 생존자는 강제 장면을 목격하고 리더를 평가할 수 있다.

- 질서와 결단을 높게 평가
- 잔혹함과 부당함으로 인식
- 자신도 다음 대상이 될 것이라 두려워함
- 거부자를 무책임하다고 비난

따라서 동일한 강제 명령도 집단 구성에 따라 다른 후속 사건을 만든다.

## 9. 행동 충돌 해결

### 9.1 방 수용 인원 초과

우선순위:

1. 플레이어 직접 명령
2. 긴급 치료·경계·복구
3. 예약된 시설 행동
4. 자율행동

밀려난 자율행동자는 다른 가능한 방과 행동을 다시 계산한다.

### 9.2 동일 설비 경쟁

전문 설비는 동시에 한 행동만 지원할 수 있다.

- 플레이어가 우선순위 지정
- 미지정 시 긴급도와 능력 적합성으로 결정
- 경쟁 자체가 관계 이벤트 조건이 될 수 있음

### 9.3 행동 담당자 부재

필요 행동을 수행할 적합한 생존자가 없으면 다음 중 하나를 선택한다.

- 낮은 능력으로 시도
- 장비·설비 보정으로 보완
- 행동 연기
- 외부 도움·세력 지원
- 위험한 임시 해결

## 10. 기본 행동 프로필

### 10.1 휴식

```yaml
action_id: HACT_REST
primary_stat: none
room_tags: [휴식]
facility_tags: [침구, 난방, 오락]
resistance: low
```

- 피로·컨디션·정신력 회복
- 방·설비·동실 인물에 따라 결과 변화
- 강한 피로 상태에서는 자율 선택 우선도가 크게 증가

### 10.2 훈련

```yaml
action_id: HACT_TRAIN
primary_stat: training_target
room_tags: [훈련]
facility_tags: [훈련_도구, 표적, 전술_자료]
resistance: medium
```

- 경험·숙련·고유 능력 해금 조건 제공
- 영구 스탯을 무제한 상승시키는 반복 행동으로 만들지 않음
- 부상·피로·기벽에 따라 거부 가능

### 10.3 대화·교류

```yaml
action_id: HACT_SOCIALIZE
primary_stat: 대화_or_매력
room_tags: [교류, 사적_대화, 집단_교류]
facility_tags: [오락, 기호품, 상담]
resistance: contextual
```

- 친밀도·스트레스·갈등 변화
- 대상의 동의와 관계 상태 필요
- 강제 교류는 일반적인 관계 회복으로 처리하지 않음

### 10.4 제작·수리

```yaml
action_id: HACT_CRAFT_REPAIR
primary_stat: 제작_or_기술
room_tags: [작업, 수리]
facility_tags: [공구, 작업대]
resistance: medium
```

- 장비 제작·수리·소비품 준비
- 부품·재료·설비 요구
- 실패 시 전체 자원 소실보다 일부 손실·추가 작업을 우선

### 10.5 간호

```yaml
action_id: HACT_CARE
primary_stat: 간호
room_tags: [치료, 상담, 회복]
facility_tags: [의료, 전문_의료, 상담]
resistance: contextual
```

- 부상·상태이상·기벽 치료
- 치료 대상이 거부할 수 있음
- 기벽별 난이도와 의약품·시설 요구 적용
- 강제 치료는 생존에 필요할 수 있으나 스트레스·불만·신뢰 손실 가능

### 10.6 경계

```yaml
action_id: HACT_GUARD
primary_stat: 탐색_or_전투
room_tags: [경계, 감시, 순찰]
facility_tags: [경보, 관측, 조명]
resistance: medium_to_high
```

- 불침번·침입 탐지·새벽 정보 보조
- 반복 배정 시 피로와 불만 누적 가능
- 전투 취약 생존자는 감시 설비로 역할 보완 가능

### 10.7 조사·연구

```yaml
action_id: HACT_RESEARCH
primary_stat: 지능_or_탐색
room_tags: [연구, 작전, 관측]
facility_tags: [지도, 기록, 분석, 통신]
resistance: low_to_medium
```

- 단서 분석·괴물 정보·지역 정보·설비 연구
- 결과는 추가 정보와 선택지 해금 중심

### 10.8 취미

```yaml
action_id: HACT_HOBBY
primary_stat: character_specific
room_tags: [오락, 개인_활동, 재배, 창작]
facility_tags: [취미, 예술, 운동, 오락]
resistance: low
```

- 스트레스 회복·관계 이벤트·고유 능력 조건
- 취미점수 등 별도 성장 자원 도입 여부는 미확정

### 10.9 시설 강화·보수

```yaml
action_id: HACT_FACILITY_WORK
primary_stat: 제작_or_기술
room_tags: [공사, 보수]
facility_tags: [공구, 건축]
resistance: medium_to_high
```

- 미개조 방의 주 개조 선택·건설
- 선택한 주 개조는 회차 동안 영구 고정
- 동일 분기의 단계 향상과 파괴 복구 가능
- 다른 주 개조로 전환 불가

## 11. R18 확장

- 공통 행동 ID를 참조해 추가 선택지와 반응을 제공한다.
- 공통 행동의 성공 여부가 R18 데이터에 의존하면 안 된다.
- R18 기벽은 행동 선호·회피·명령 저항에 영향을 줄 수 있다.
- R18 전용 강제 행동은 공통 강제 명령과 분리된 모듈 규칙이 소유한다.
- R18 OFF에서는 해당 확장 반응과 태그를 무시한다.

- 아침·저녁 실제 행동 횟수
