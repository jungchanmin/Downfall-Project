# 📦 자원 체계 시스템

---

## SECTION 1 — 자원 체계 개요

Downfall의 보상과 소지품은 데이터 관리 방식에 따라 세 계층으로 구분한다.

| 계층 | 항목 | 관리 방식 |
|---|---|---|
| **수량형 자원** | 식량 / 부품 / 탄약 / 금화 | 파티 공용 정수값 |
| **개별 아이템** | 장착 장비 / 사용 아이템 / 중요 물품 / 유물 | 종류별 개별 데이터 |
| **진행·사회 자원** | 명성 / 단서 / 세력평판 / 동료 | 수치·카드·플래그·NPC별 관리 |

### 1-1. 설계 원칙

- 탐색 중에는 구체적인 물건과 발견 장면을 보여준다.
- 귀환 정산에서는 반복 관리가 필요한 물자를 단순한 수량형 자원으로 환산할 수 있다.
- 제작은 핵심 콘텐츠가 아니므로 재료를 세분화하지 않는다.
- 장비는 사용처보다 **보유·사용 방식**을 기준으로 분류한다.
- 출발 전에 배정하지 않은 장비는 외출 중 사용할 수 없다.
- 연료와 전력은 독립 자원으로 관리하지 않는다.

## SECTION 2 — 수량형 자원

### 2-1. 식량 (Food)

```text
관리 방식: 파티 공용 정수값
기본 소비: 생존자 1명당 한 끼 1포인트
```

**역할**: 아지트 생존자의 허기를 관리하는 핵심 자원.

#### 탐색과 환산

- 탐색 중에는 통조림, 빵, 건조식량 등 개별 식품으로 묘사한다.
- 아지트 귀환 정산 시 개별 식품을 식량 포인트로 환산한다.
- 식품별 환산량은 Item DB에서 관리한다.
- 환산을 마친 일반 식품은 개별 인벤토리에서 제거하고 파티 식량에 합산한다.

#### 현장 즉석 소비

- 탐색 중 발견한 식품은 즉시 섭취할 수 있다.
- 즉석 섭취는 해당 캐릭터의 허기를 회복하는 현장 보상이다.
- 즉시 섭취한 식품은 귀환 시 식량 포인트로 환산되지 않는다.
- 오염·부패·특수 식품의 부작용은 개별 Item 또는 Event 데이터가 소유한다.

#### 부족 상태

- 식량이 부족하면 해당 끼니의 정상 배식이 불가능하다.
- 배식 실패 시 허기 악화와 후속 체력·불만 이벤트가 발생할 수 있다.
- 정확한 허기 변화량은 NPC Stats 및 밸런스 문서가 소유한다.

---

### 2-2. 부품 (Parts)

```text
관리 방식: 파티 공용 정수값
세부 재료 분리: 사용하지 않음
```

**역할**: 제작, 수리, 시설 보강에 사용하는 통합 재료.

#### 사용처

- 무기·방어구·유틸리티 장비 수리
- 제한적인 장비 제작 또는 개조
- 아지트 수리와 방어 설비 구축
- 이벤트에서 요구하는 기계·전자·화학 재료의 통합 비용

목재, 금속, 전자부품, 화학재료는 개별 자원으로 나누지 않는다. 서술상 구체 물질이 등장하더라도 시스템상 `부품`으로 환산한다.

---

### 2-3. 탄약 (Ammunition)

```text
관리 방식: 파티 또는 외출조 공용 정수값
탄종 구분: 사용하지 않음
```

**역할**: 모든 총기류가 공통으로 소비하는 전투 자원.

#### 핵심 규칙

- 총기별로 1회 공격 시 탄약 소비량이 다를 수 있다.
- 총기 내구도와 탄약은 별도로 관리한다.
- 탄약이 없으면 해당 총기는 사용 불가하거나 제한된 비사격 행동만 가능하다.
- 구체 소비량과 재장전 규칙은 Combat 및 Item DB가 소유한다.

---

### 2-4. 금화 (Gold)

```text
관리 방식: 파티 공용 정수값
주요 사용처: 암시장·특수 거래·뇌물·특수 서비스
```

**역할**: 일반 생활 화폐가 아니라 위험한 시장과 특수 거래에서 통용되는 희귀 화폐.

- 일반 생존자 사이의 거래는 물물교환 또는 이벤트 선택지로 처리한다.
- 암시장과 일부 세력 서비스는 금화를 요구한다.
- 금화 가치와 거래 가격은 Economy 또는 Event 데이터가 소유한다.

## SECTION 3 — 개별 아이템

개별 아이템은 `Wiki/04_Items/ITEM_System_Overview.md`와 세부 DB가 소유한다.

### 3-1. 장착 장비

캐릭터가 손에 들거나 착용하여 반복 사용하는 아이템.

| 분류 | 역할 |
|---|---|
| **무기** | 근접 / 원거리 / 투척·폭발 / 임시 무기 |
| **보호 장비** | 일반 방어 / 환경 보호 / 특수 보호 |
| **유틸리티 장비** | 여러 탐색·수리·의료·관측 행동에 범용 보정과 일부 특별 행동 제공 |

#### 내구도와 사용 횟수

- 근접 무기, 총기, 방어구 등은 내구도를 사용할 수 있다.
- 공구세트, 의료가방, 필터·배터리 장비 등은 사용 횟수를 사용할 수 있다.
- 어떤 방식을 쓸지는 아이템별 데이터가 결정한다.
- 모든 장비에 동일한 내구도 체계를 강제하지 않는다.

### 3-2. 사용 아이템

사용하면 소모되거나 사용 횟수가 감소하는 아이템.

| 분류 | 역할 |
|---|---|
| **의료품** | 치료, 응급처치, 상태이상 대응 |
| **전술품** | 전투·탐색·탈출에서 창의적으로 사용하는 일회성 도구 |
| **기호품** | 스트레스·컨디션·관계·거래에 활용 |
| **특수 소비품** | 일반 분류에 속하지 않는 사건·의식·환경 대응 아이템 |

#### 의료품 하위 분류

| 분류 | 기준 |
|---|---|
| **일반 의료품** | 비전투 상황에서 누구나 사용할 수 있는 기본 치료 |
| **전문 의료품** | 고성능 효과. 간호 능력 또는 사용 조건 요구 |
| **응급 의료품** | 전투 중 즉시 사용 가능한 응급처치 |
| **상태이상 치료품** | 감염·중독·화상 등 특정 상태이상 대응 |

### 3-3. 중요 물품

열쇠, 출입증, 지도, 문서, 암호, 세력 증표, 개인 유품과 퀘스트 물품.

- 일반적으로 소비하지 않는다.
- 별도 중요 물품 목록 또는 진행 플래그로 관리할 수 있다.
- 기본적으로 일반 휴대 슬롯을 차지하지 않는다.
- 이벤트가 명시한 경우에만 소비·분실될 수 있다.

### 3-4. 유물

강력한 효과와 세계관적 의미를 가진 특수 아이템.

#### 공통 원칙

- 일반 제작으로 생산할 수 없다.
- 획득 과정이 어렵거나 위험하다.
- 강력한 효과와 함께 대가 또는 부작용이 존재할 수 있다.
- 의식, 어머니, 괴물, 인물의 기억 등 핵심 로어와 연결될 수 있다.

#### 사용 방식

유물을 모두 무제한 귀속형으로 고정하지 않는다.

- 지속형
- 장착형
- 능동 사용형
- 충전형
- 소모형
- 저주형

귀속·거래·수리·파괴 가능 여부는 유물별 데이터가 결정한다.

## SECTION 4 — 장비 배정과 인벤토리

### 4-1. 아지트 보관

- 획득한 일반 장비와 사용 아이템은 파티 공용 창고에 보관한다.
- 식량·부품·탄약·금화는 파티 공용 수치로 관리한다.
- 중요 물품과 단서는 별도 목록으로 관리할 수 있다.

### 4-2. 외출 준비

- 외출 전에 캐릭터별 장착 장비와 휴대품을 배정한다.
- 캐릭터 장착 슬롯과 파티 공용 휴대 가방을 함께 사용한다.
- 장소의 위험도, 거리, 임무와 캐릭터 역할에 따라 준비가 달라져야 한다.

### 4-3. 외출 중 제약

- 배정하지 않은 창고 아이템은 외출 중 사용할 수 없다.
- 외출 중 획득한 아이템은 휴대 한도 안에서 보관한다.
- 휴대 한도를 넘으면 버리기, 즉시 사용, 교체, 분담 중 하나를 선택해야 한다.
- 구체 슬롯 수와 무게 규칙은 Inventory 설계 단계에서 확정한다.

### 4-4. 유틸리티 장비 사용

유틸리티 장비는 한 가지 고정 행동만 수행하는 열쇠가 아니다.

- 관련 판정에 보정을 제공한다.
- 일부 특별 행동이나 선택지를 해금할 수 있다.
- 실제 허용 효과는 이벤트와 아이템 데이터 양쪽에서 확인한다.

예: 공구세트는 제작, 수리, 해체, 함정 해제와 시설 조사에 활용될 수 있다.

## SECTION 5 — 진행·사회 자원

### 5-1. 명성 (Renown)

생존자 무리의 사회적 위상과 업적을 나타내는 내부 수치.

- 고위험 구역 생환
- 멸망의 괴물 관련 업적
- 세력 대형 의뢰
- 특별한 생존·탐색 성과

명성 단계와 엔딩·어머니 평가 연동은 별도 정본에서 확정한다.

### 5-2. 단서 (Clue)

단서는 범용 카드와 고유 정보의 혼합 구조를 사용한다.

#### 일반 단서 카드

- 판정 보정
- 숨겨진 선택지 개방
- 세력 협상
- 지역 위험 또는 괴물 정보 공개

여러 용도로 사용할 수 있으나, 각 이벤트가 허용 사용처와 소비 여부를 명시한다.

#### 고유 단서

- 이름과 출처가 있는 특정 정보
- 장소, 인물, 괴물, 의식 또는 보물찾기와 연결
- 관련 이벤트·루트·보관함·접근 권한을 해금
- 기본적으로 소비하지 않지만 이벤트가 명시하면 소모 가능

### 5-3. 세력평판 (Faction Reputation)

세력별 관계 상태를 관리한다.

- 정확한 범위와 임계값은 Faction System에서 확정한다.
- 거래, 진입 허가, 의뢰, 지원, 적대 전투에 영향을 준다.

### 5-4. 동료 (Survivor)

동료는 수치형 물자가 아니라 개별 NPC로 관리되는 인적 자원이다.

- 초기 동료와 신규 합류 인물 모두 개별 캐릭터 데이터가 필요하다.
- 합류 이후 관계, 불만, 충성, 적응과 역할 분담의 대상이 된다.

## SECTION 6 — 이벤트 및 장소 연동 원칙

### 6-1. 획득과 소비

- 실제 획득·소비량은 각 Event 또는 Item 데이터가 소유한다.
- 장소 문서는 획득 가능한 **분류 후보**만 기록한다.
- 구체 아이템명은 정식 Item DB에 존재하는 항목만 사용한다.

### 6-2. 연료·전력 처리

연료와 전력은 독립 자원으로 관리하지 않는다.

- 차량, 발전기, 전력 복구는 개별 이벤트 조건으로 처리한다.
- 필요한 비용은 부품, 장비 사용 횟수 또는 특정 중요 물품으로 표현한다.

### 6-3. 1회 변동 상한

현재 수치는 임시값이며 밸런스 단계에서 조정한다.

```text
식량: ±3
부품: ±3
탄약: 이벤트·전투 기준 임시
금화: ±5
명성: +1 ~ +10
세력평판: ±10
```

장비, 유물, 단서와 동료는 개수 또는 개별 데이터 단위로 관리한다.

---

## SECTION 7 — 비자원 항목

기벽, 상태이상, 관계 스탯과 침식은 자원 체계와 혼용하지 않는다.

| 항목 | 관리 위치 |
|---|---|
| 기벽 | Quirk DB 및 NPC 상태 |
| 상태이상 | Status Effect DB |
| 친밀도·불만·적응 | NPC Stats 및 관계 시스템 |
| 침식 | NPC Stats 또는 R18/의식 관련 시스템 |

## SECTION 8 — 미확정 항목

다음은 별도 설계·DB 작업 전까지 확정하지 않는다.

- 세부 식품과 식량 환산값
- 개별 무기·방어구·유틸리티 장비
- 개별 의료품·전술품·기호품
- 장비 슬롯 수와 휴대 한도
- 내구도·사용 횟수의 구체 수치
- 총기별 탄약 소비량
- 개별 유물과 사용 방식
- 일반 단서 카드의 보유 상한과 소비 규칙
