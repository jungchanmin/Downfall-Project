# R18 굴복 페이즈 코어 교환 규칙 v1.0

## 0. 시스템 정체성

굴복 페이즈는 단순한 기술 피해 교환이 아니다.

```text
커맨드 상성
+
주도권
+
자세·부위·행동 점유
+
상태이상 콤보
+
불완전한 공격 전조
```

가 결합된 합 단위 대전 시스템이다.

플레이어의 핵심 판단은 다음과 같다.

- 상대의 공격축과 기술을 얼마나 정확히 읽었는가
- 현재 자세와 점유 상태에서 실제로 사용할 수 있는 커맨드가 무엇인가
- 이번 합에서 피해를 줄일지, 카운터로 주도권을 탈취할지, 저항으로 상태를 정리할지
- 주도권을 유지해 콤보를 이어갈지, 안전하게 넘길지

---

## 1. 양측 커맨드 체계

### 1-1. 괴물 공격 커맨드 6축

| 축 | 전투 역할 | 장기 기록 |
|---|---|---|
| Contact | 접촉·견제·부위 접근 | Contact |
| Penetration | 자세·부위 돌파·강제 수용 | Penetration |
| Fluid | 체액 공격·의존·조건부 자원 압박 | Fluid |
| Mutation | 신체 개조·변이·영구 변화 위험 | Mutation |
| Cognition | 정신·인식·판단·전조 왜곡 | Cognition |
| Subjugation | 의지 파괴·결정타·피니셔 | Subjugation |

`Subjugation`은 일반 견제축보다 결정타 성격이 강하다. 함락 기술은 굴복도 임계, 특정 상태이상, 자세, 주도권 또는 점유 조건을 요구하는 것을 기본으로 한다.

### 1-2. 생존자 대응 커맨드 5분류

| 분류 | 코어 역할 |
|---|---|
| Service | 저부담 능동 공세·견제 |
| Tease | 고위험 패링·카운터·즉시 주도권 탈취 |
| Domination | 고위력 역전·강공·마무리 |
| Absorption | 저효율 회복·특수상황 증폭·양날 자원 전환 |
| Defiance | 수비·상태 해제·안정적 탈출·다음 경합 준비 |

기벽 Return은 이 기본 역할을 대체하지 않는다. 기벽은 분류의 특정 상황, 추가 효과, 변종과 Risk 교환을 강화한다.

---

## 2. 주도권

주도권은 단순 선공권이 아니다.

```text
선행 커맨드권
+
자세 결정권
+
부위·행동 점유권
+
콤보 유지권
```

주도권 보유자는 공격 의도를 생성하고, 기술 조건에 따라 자세를 강제하거나 부위·행동을 점유한다.

피주도권자는 전조 정보를 받은 뒤 대응 커맨드를 선택한다.

---

## 3. 공격 전조와 정보 정확도

### 3-1. 전조 생성

전조 정보는 다음 요소로 생성한다.

- 탐색·지능·관련 능력
- 괴물 유형 지식
- 과거 조우 기록
- 동일 기술 관찰 횟수
- 관련 Memory Log
- 현재 피로·탈진·정신 상태
- 최면·홀림·혼란·인지 공격

### 3-2. 정보 단계

| 단계 | 표시 범위 |
|---|---|
| none | 전조 감지 실패 |
| vague | 복수 축 후보 또는 모호한 방향 |
| axis | 공격축 공개 |
| axis_part | 공격축 + 목표 부위 공개 |
| detailed | 실제 기술군·자세 강제·점유 후보 공개 |

### 3-3. 정보 신뢰도

일반 관찰 실패나 숙련 부족은 정보의 불확실성을 플레이어에게 알린다.

```text
[예상 공격: Contact 또는 Cognition]
[정보 신뢰도: 낮음]
```

최면·환각·인지 교란처럼 명확한 특수 원인이 있을 때만 거짓 전조를 확정 정보처럼 표시할 수 있다.

```text
실제: Penetration + LowerBody
표시: Contact + Chest
```

거짓 전조는 무작위 상시 페널티가 아니라 상태이상·기술 effect가 소유한다.

### 3-4. 지식과 기록의 보상

동일 괴물·기술을 반복 관찰하면 다음 혜택 후보를 얻는다.

- 최소 전조 단계 상승
- 신뢰도 표시 개선
- 특정 거짓 전조 간파
- 시그니처 기술 후보 축소
- 지원 생존자의 전조 교정 행동 해금

---

## 4. 합 단위 처리 순서

```text
1. 현재 주도권 확인
2. 주도권자 커맨드·기술·목표 부위·자세·점유 결정
3. 공격 전조 생성
4. 피주도권자 대응 커맨드 선택
5. 자세·부위·행동 게이트 검사
6. 상성 검사
7. 능력·경험·기벽·로그·상태 보정 판정
8. 게이지·피해·상태·로그·점유 적용
9. 주도권 처리
10. 상태이상 내성 갱신
11. 지원·교대 창 확인
```

게이트 검사가 상성 판정보다 먼저다. 상성이 유리해도 현재 자세나 점유 때문에 행동 자체가 불가능하면 커맨드는 취소된다.

---

## 5. 자세·부위·행동 게이팅

### 5-1. 게이트 종류

- 자세 게이트: 현재 자세에서 기술 사용 가능 여부
- 대상 부위 게이트: 목표 부위 접근 가능 여부
- 행동 부위 게이트: Mouth·Hand·LowerBody 등 시전자 행동 부위 사용 가능 여부
- 점유 게이트: 상대가 점유한 부위·행동과 충돌하는지
- 상태 게이트: 구속·속박·절정 여운 등으로 대응 자체가 제한되는지

### 5-2. 행동 취소

피주도권자의 대응이 게이트에 걸리면:

```text
대응 커맨드 취소
→ 주도권자 커맨드 일방 적용
→ 기본적으로 피주도권자에게 주도권 이전
```

이는 주도권자가 게이팅 설계로 큰 보상을 얻되, 기본 규칙만으로 무한 콤보를 이어가지 못하게 하는 안전장치다.

### 5-3. 주도권 유지 예외

기술·상태이상에 `initiative_hold` 또는 동등 효과가 있으면 행동 취소 뒤에도 주도권을 유지할 수 있다.

```yaml
initiative_rule:
  on_gate_cancel: pass_to_defender
  hold_effect_tags:
    - initiative_hold
    - combo_continue
```

주도권 유지 effect는 명시적으로 기술이나 상태이상이 소유한다.

---

## 6. 상성·카운터

### 6-1. 역상성 카운터

피주도권자가 상대 공격축을 정확히 찌르는 역상성 카운터에 성공하면 반드시 주도권을 획득한다.

```text
일반 성공
→ 상대 공격 무효 또는 대폭 약화
→ 카운터 측 주도권 획득

큰 성공
→ 주도권 획득 + 자세 해제·역상태이상 후보

대성공
→ 공격 역이용 + 유리 자세·점유 후보
```

카운터 실패는 일반 대응보다 더 큰 손해를 받을 수 있다.

### 6-2. 중립 상성

중립 상성은 자동 승패가 아니다. 커맨드 기본 위력, 능력, 경험, 기벽, 로그, 자세, 점유, 상태와 주도권 보정을 비교한다.

| 결과 차이 | 처리 |
|---|---|
| 큰 우세 | 상대 행동 취소·효과 적용·주도권 획득 또는 유지 후보 |
| 근소 우세 | 양측 일부 적용, 우세측 효과 강화 |
| 동률 | 양측 일부 적용, 자세·점유 유지, 강한 상태이상 제한 |
| 근소 열세 | 자신의 효과 약화, 상대 일부 적용 |
| 큰 열세 | 행동 취소, 상대 전부 적용 |

정확한 수치와 결과 차이 임계는 밸런스 단계에서 확정한다.

## 7. 생존자 5분류 전술 계약

### 7-1. Service

- 능동 공세
- 낮은 자기 부담
- 안정적인 견제
- 상대 자세 강제·점유 준비 방해 후보
- 강한 즉시 역전과 대량 피해는 제한

### 7-2. Tease

- 현재 들어오는 공격을 받아치는 즉응형 패링
- 성공 시 확정 주도권 탈취
- 실패 시 상대 공격 증폭·점유 강화 가능
- 고위험 고리턴

### 7-3. Domination

- 역착정·기승·역제압형 강공
- 높은 흥분·탈진·굴복 압박
- 성공 시 유리 자세와 주도권 유지 후보
- 실패 시 함락·결정타에 노출되는 하이리스크

### 7-4. Absorption

- 평시 낮은 효율
- Fluid 공격 대응 또는 특정 상태에서 효과 증폭
- 회복·자원 전환과 동시에 Fluid·Mutation·의존 Risk 증가
- 무료 회복이 아니라 상황형 양날 기술

### 7-5. Defiance

- 상태이상 해제
- 굴복도·피해 경감
- 자세·점유 탈출
- 상태이상 내성 획득량 증가 후보
- 상태 해제와 주도권 획득은 별개
- 다음 경합을 유리하게 만드는 안전판

## 8. 상태이상·콤보

### 8-1. 상태 분류

| 분류 | 역할 |
|---|---|
| 행동 방해형 | 비용·성공률·선택 정확도 저하 |
| 정보 교란형 | 전조 누락·신뢰도 저하·거짓 전조 |
| 게이팅형 | 자세·부위·행동 사용 제한 |
| 콤보 유지형 | 행동 취소 후 주도권 유지 |
| 결정타 게이트형 | Subjugation·피니셔 기술 조건 제공 |

### 8-2. 연속 주도권 상한

주도권 유지 effect가 있어도 기본 연속 상한을 사용한다.

```yaml
combo_state:
  chain_count: 0
  chain_limit: TBD
```

- 주도권 유지 성공 시 `chain_count +1`
- 상한 도달 시 상태이상이 남아 있어도 다음 합 종료 후 주도권 이전
- 보스·특수 괴물·피니셔 effect만 상한 확장 가능
- 정확한 기본 상한은 밸런스 테스트 후 확정

### 8-3. 중첩 제한

하나의 합에서 여러 `initiative_hold` effect가 중첩되어 추가 합을 각각 생성하지 않는다.

```text
여러 유지 effect 보유
→ 1회 유지 판정 또는 1회 유지 소비
→ 나머지는 성공률·게이트 강도·상태 지속 보조
```

---

## 9. 상태이상 내성

상태 해제 또는 자연 종료 후 동일 상태 내성이 상승한다.

```text
상태 부여
→ 노출 기록

상태 해제·종료
→ 동일 상태 내성 상승

재부여 시도
→ 성공률·지속시간·게이트 강도 감소
```

Defiance 성공은 내성 상승 또는 다음 주도권 경합 보정을 제공할 수 있지만, 상태 해제만으로 즉시 주도권을 얻지는 않는다.

---

## 10. 교대·지원 초안

기본 활성 전투는 1대1을 사용한다.

```text
괴물 1체
vs
활성 생존자 1명
+
대기 생존자 지원
```

대기 생존자 후보 행동:

- 전조 정보 교정
- 상태이상 해제 보조
- 점유 해제 보조
- 주도권 교란
- 교대 요청
- 긴급 구조

다대다 동시 커맨드 교환은 기본 범위에서 제외한다.

교대 시 개인 상태는 유지한다. 자세·점유·주도권의 세부 승계는 후속 태그 배틀 설계에서 확정한다.

---

## 11. 기벽 Return 연결 원칙

기벽은 기본 커맨드 역할을 대체하지 않고 전투 처리 단계 중 하나를 강화한다.

- Service: 저부담 견제, 전조 보조, 점유 약화
- Tease: 거짓 전조 간파, 카운터 보상, 실패 비용 조정
- Domination: 고위력 역전, 자세 강제, 주도권 유지와 큰 실패 Risk
- Absorption: Fluid 대응 증폭, 회복과 변질 부채
- Defiance: 상태 해제, 내성, 탈출, 다음 경합 보정

기벽 완화 중에는 기벽 전용 Return과 변종만 비활성화하고 기본 커맨드는 유지한다.
