# 기벽 치료·완화 공통 시스템

## 1. 목적

이 문서는 일반 기벽과 R18 기벽을 동일한 치료 규칙으로 처리하지 않도록 경계를 정의한다.

- 일반 기벽은 생존 난이도와 행동 가능성을 직접 악화하므로 제거 경로를 충분히 제공한다.
- R18 기벽은 R18 메모리 로그와 연결되는 Risk & Return형 장기 변화이므로 원칙적으로 제거하지 않는다.
- 공통 간호 행동과 의료 설비는 두 계열에 서로 다른 방식으로 작동한다.

## 2. 기벽 계열

### 2.1 일반 기벽

육체적 문제:

- 신체 외부
- 신체 내부

정신적 문제:

- 정신이상
- 욕구폭발·충동
- 트라우마

일반 기벽은 명확한 패널티와 생존 부담을 제공하며, 게임 진행 중 제거 또는 치료할 수 있어야 한다.

### 2.2 R18 기벽

육체 계열:

- 신체 개조
- 감도 개발
- 내구성

정신 계열:

- 성벽
- 상식 개변
- 욕구

R18 기벽은 단순한 질병이 아니라 반복된 R18 경험과 타락 방향이 인물에게 남긴 변화다.

## 3. 공통 메타데이터

모든 기벽 DB는 다음 필드를 사용한다.

```yaml
quirk_treatment:
  treatment_model: removable | suppressible | self_expiring | irreversible
  treatment_grade: easy | normal | hard | professional | special | none
  required_items: []
  required_facilities: []
  required_stats: []
  special_conditions: []
  failure_cost: []
  partial_success: null
```

### 3.1 treatment_model

| 값 | 의미 |
|---|---|
| `removable` | 치료 성공 시 기벽 제거 |
| `suppressible` | 제거되지 않지만 효과·단계·지속시간 완화 가능 |
| `self_expiring` | 정해진 지속시간 후 자연 해제, 치료로 단축 가능 |
| `irreversible` | 일반 치료로 제거·완화 불가, 특수 사건만 예외 가능 |

### 3.2 treatment_grade

| 등급 | 일반 요구 방향 |
|---|---|
| 쉬움 | 간호 판정 |
| 보통 | 간호 + 일반 의료품 또는 적합한 기호품 |
| 어려움 | 높은 간호 + 전문 의료품 또는 적합한 시설 |
| 전문적 치료 | 전문 의료품 + 높은 간호 + 의무실·전문 설비 |
| 특수 | 이벤트·유물·고유 능력·희귀 설비 |
| 없음 | 일반 치료 대상이 아님 |

## 4. 일반 기벽 치료 원칙

### 4.1 제거 가능성

일반 기벽은 원칙적으로 `removable`이다.

예외:

- 설정상 신체가 완전히 상실되거나 복구 불가능한 경우
- 고유 NPC 서사와 직접 연결된 기벽
- 특수 저주·유물·괴물 효과

예외 기벽도 완전 무대응 상태로 두지 않고, 패널티 완화·보조 장비·행동 우회 경로를 제공한다.

### 4.2 한 번의 치료 행동

기본적으로 장기 진행도 게이지를 사용하지 않는다.

```text
치료 조건 확인
→ 간호 판정
→ 성공 / 부분 성공 / 실패
```

- 성공: 기벽 제거
- 부분 성공: 일정 기간 패널티 완화, 다음 시도 보정 또는 자원 일부 보존
- 실패: 행동 소모와 일부 자원 손실

### 4.3 치료 접근성

일반 기벽의 제거 수단은 하나로 고정하지 않는다.

가능한 경로:

- 간호 스탯
- 일반 의료품
- 전문 의료품
- 의무실·상담실·회복실
- 특정 생존자 고유 능력
- 이벤트 보상
- 적합한 기호품 또는 관계 행동

같은 기벽도 파티 구성과 아지트 빌드에 따라 다른 치료 경로를 가질 수 있다.

## 5. R18 기벽 운용 원칙

### 5.1 기본 비가역성

R18 기벽은 원칙적으로 제거 대상이 아니다.

- 메모리 로그의 누적 기록과 해금 결과를 보존한다.
- 플레이어가 R18 경험의 대가와 이득을 함께 받아들이도록 한다.
- 일반 간호·의약품으로 캐릭터의 누적 변화를 초기화하지 않는다.
- 회차 중 자유로운 제거·재획득 반복을 금지한다.

### 5.2 Risk & Return

R18 기벽은 순수 패널티가 아니라 다음 구조를 가져야 한다.

```text
이득 또는 특수 선택지
+
행동 제한·관계 변화·침식 위험·고착
```

설계 예:

- 특정 굴복 기술 강화 + 저항 경로 약화
- 스트레스 면제 + 위험한 조우 선택지 강제
- 관계 행동 강화 + 공공 행동 패널티
- 회복 효과 + 침식도 상승

순수한 상위 호환 보너스나 순수한 일반 기벽 복제는 피한다.

### 5.3 완화 가능한 R18 기벽

다음은 `suppressible` 또는 `self_expiring`으로 지정할 수 있다.

- 지속시간형 욕구
- 단계형 감도 개발
- 일시적 상식 개변
- 특정 R18 사건이 유지하는 상태

완화 수단:

- 메모리 로그의 반대 방향 기록
- 특정 해금 특성
- R18 전용 상담·회복 이벤트
- R18 전용 설비
- 지속시간 경과

완화는 다음 중 하나다.

- 단계 1 하향
- 일정 페이즈 효과 억제
- 지속시간 감소
- 패널티 일부 비활성

기벽 자체와 메모리 기록은 삭제하지 않는다.

### 5.4 비가역 R18 기벽

다음은 기본적으로 `irreversible`이다.

- 영구 신체 변이
- 영구 성벽 고착
- 영구 상식 개변
- 메모리 로그의 최종 해금과 결합된 변화

특수 이벤트가 존재하더라도 일반 치료가 아니라 별도의 서사적 선택으로 취급한다.

예외 제거는 다음 조건을 가져야 한다.

- 희귀하고 명시적인 사건
- 새로운 대가 또는 다른 기벽 발생 가능
- 메모리 로그 기록은 유지
- 플레이어가 회차 빌드를 쉽게 초기화하는 수단이 되지 않음

## 6. 메모리 로그 연결

R18 기벽의 획득·완화·고착은 `MECH_R18_Memory_Log`가 기록한다.

```yaml
r18_quirk_meta:
  unlock_condition:
  memory_log_links: []
  risk_effects: []
  return_effects: []
  mitigation_model: none | duration_reduction | stage_down | temporary_suppression
  lock_state: temporary | persistent | permanent
```

### 6.1 기록 보존

R18 기벽이 완화되거나 지속시간이 끝나도 다음은 유지한다.

- 경험 카운터
- 해금 특성
- 주요 기억 플래그
- 과거 기벽 획득 기록

따라서 인물은 완전히 이전 상태로 되돌아간 것이 아니라 현재 효과만 억제된 상태다.

### 6.2 고착

일정 임계값 또는 반복 획득 시 다음 변화가 가능하다.

```text
일시 기벽
→ 지속 기벽
→ 고착 기벽
→ 영구 변화
```

고착 기준은 각 기벽 또는 메모리 로그 해금 테이블이 소유한다.

## 7. 아지트 행동과 시설

### 7.1 일반 기벽

- 의무실: 육체 기벽
- 상담실: 정신이상·트라우마
- 회복실: 중증 회복과 치료 후 안정
- 제독실: 오염·감염 계열
- 전문 의료장비: 전문적 치료

### 7.2 R18 기벽

공통 시설은 R18 기벽을 제거하지 않는다.

R18 ON일 때 모듈은 같은 방 ID를 참조해 다음을 추가할 수 있다.

- 일시 억제
- 지속시간 단축
- 욕구·침식 안정
- 메모리 로그 기반 역행 이벤트
- 특정 R18 기벽의 Risk 또는 Return 조정

## 8. UI 표시 원칙

일반 기벽:

```text
치료 가능
치료 난이도
필요 아이템
필요 시설
```

R18 기벽:

```text
지속 상태
완화 가능 여부
고착 단계
Risk 효과
Return 효과
관련 메모리 로그
```

R18 기벽을 일반 질병처럼 `치료 가능`으로 표시하지 않는다.
