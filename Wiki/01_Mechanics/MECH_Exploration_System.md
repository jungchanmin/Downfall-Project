# 외출·탐험 시스템

## SECTION 1 — 목적과 설계 원칙

### 1-1. 목적

외출·탐험은 생존자 편성, 장비 준비, 장소 선택, 필수 시련, 괴물 조우, 자원 획득과 귀환 판단을 하나의 반복 가능한 플레이 흐름으로 연결한다.

### 1-2. 제한된 자유도

Downfall의 탐험은 완전 자유 입력이 아니라 **제한된 선택의 연속**을 사용한다.

- 플레이어는 목적지, 외출조, 장비, 접근 방식, 탐색 행동과 귀환 시점을 결정한다.
- 한 장면의 기본 선택지는 보통 2~4개로 제한한다.
- 생존자 스탯, 장비 태그, 단서와 상태에 따라 추가 선택지가 열린다.
- 모든 아이템을 모든 사건에 임의로 사용할 수는 없다.
- 이벤트가 허용한 기능 태그와 상황 조건을 기준으로 선택지를 생성한다.

### 1-3. 상위 플레이 원칙

- 외출은 점심 페이즈에만 수행한다.
- 한 외출은 반드시 해당 지역의 필수 시련을 포함한다.
- 해당 지역에 맵 배치 괴물이 존재하면 반드시 조우한다.
- 전투, 도주, 협상, 은폐, 장비 활용 등 해결 방식은 사건과 조건에 따라 달라진다.
- 플레이어는 필수 시련을 결론짓기 전에는 정상 귀환할 수 없다.
- 위험을 감수하고 더 깊이 탐색할수록 보상과 손실 가능성이 함께 증가한다.

## SECTION 2 — 외출 8단계 흐름

### 2-1. 목적지 선택

지도에서 방문할 지역을 선택한다.

플레이어에게 표시 가능한 정보:

- 지역 위험도
- 잔존 자원도
- 주요 획득 카테고리
- 알려진 세력
- 새벽에 배치된 괴물
- 초회 지역 이벤트 완료 여부
- 발견한 세부 구역
- 관련 고유 단서
- 접근 제한과 알려진 우회로

정보는 탐색, 지능, 지도, 단서와 이전 방문 결과에 따라 일부만 공개될 수 있다.

### 2-2. 외출조 편성

외출 전에 다음을 결정한다.

- 참여 생존자
- 역할
- 무기
- 보호 장비
- 유틸리티 장비
- 사용 아이템
- 공용 가방
- 공용 탄약

아지트 창고에 남겨둔 아이템은 외출 중 사용할 수 없다.

### 2-3. 접근

지역 진입 전 접근 방식을 선택한다.

대표 선택지:

- 정면 접근
- 우회
- 정찰
- 신속 진입
- 단서 기반 비밀 경로

접근 결과는 초기 경계도, 첫 조우 위치, 기습 여부, 피로와 시간 손실에 영향을 준다.

### 2-4. 맵 배치 괴물 조우

해당 지역에 맵 배치 괴물이 존재하면 반드시 조우 단계를 처리한다.

가능한 대응:

- 전투
- 도주
- 은폐
- 유인
- 장소 장치 활용
- 단서·장비·스탯 기반 특수 대응

괴물과 지역 이벤트의 처리 순서는 장소별 `mandatory_sequence`가 정의한다.

### 2-5. 지역 필수 이벤트

각 지역은 해당 장소의 성격을 보여주는 지역 필수 이벤트를 가진다.

- 첫 방문: 초회 지역 이벤트 우선
- 이후 방문: 반복 지역 이벤트군 중 하나
- 필수 이벤트를 결론짓기 전 정상 귀환 불가

완료는 성공만을 뜻하지 않는다.

- 성공
- 부분 성공
- 실패
- 대가 지불
- 추격 상태로 종료
- 사건 결과를 감수하고 후퇴

무한 봉쇄를 방지하기 위해 사건이 결과 상태에 도달하면 완료로 본다.

### 2-6. 자유 탐색

필수 시련 이후 자원과 단서를 찾기 위한 선택 단계다.

대표 행동:

- 빠르게 뒤진다
- 신중하게 조사한다
- 특정 자원 카테고리를 집중 탐색한다
- 주변을 경계한다
- 잠긴 공간을 연다
- 고유 단서를 추적한다
- 심층 구역에 진입한다

탐색 행동 횟수와 세부 비용은 후속 밸런스 단계에서 확정한다.

### 2-7. 추가 조우와 심층 진행

자유 탐색 중 다음 결과가 발생할 수 있다.

- 자원 발견
- 지역 환경 위험
- 랜덤 괴물
- 맵 배치 괴물 재조우
- 세력·생존자 조우
- 숨은 구역
- 고유 사건
- 아무것도 없음

각 결과 후 플레이어는 계속 탐색, 다른 구역 이동, 심층 진입 또는 귀환을 선택한다.

### 2-8. 귀환

귀환 단계에서는 다음이 작동할 수 있다.

- 과적
- 추격
- 길 잃음
- 부상·기벽 패널티
- 장비 파손
- 확보한 우회로
- 동료 지원

귀환 성공 후 식품 환산, 획득물 정산, 괴물 상태 저장과 지역 잔존 자원도 변화를 처리한다.

## SECTION 3 — 지역 필수 이벤트

### 3-1. 정의

지역 필수 이벤트는 단순 랜덤 사건이 아니라 장소를 통과하기 위한 시련이다.

장소 문서는 다음을 기록한다.

```yaml
region_trial:
  first_visit_event:
  repeat_event_pool: []
  mandatory: true
  retreat_rule:
  failure_result:
```

### 3-2. 설계 원칙

- 장소의 핵심 위협과 분위기를 보여준다.
- 단순 전투만 반복하지 않는다.
- 환경, 세력, 구조, 괴물, 윤리적 선택을 사용할 수 있다.
- 실패해도 외출 전체가 영구 정지하지 않도록 결론 상태를 제공한다.
- 초회 사건은 장소 정보와 향후 탐색 구조를 해금할 수 있다.

### 3-3. 귀환 제한

플레이어는 지역 필수 이벤트를 결론짓기 전에 정상 귀환할 수 없다.

단, 다음 특수 결과는 가능하다.

- 강제 퇴각
- 추격 상태 귀환
- 동료 또는 장비를 잃고 탈출
- 사건을 미해결 상태로 남기되 심각한 대가를 수용

이 경우에도 이벤트는 명시적인 결과 상태를 기록한다.

## SECTION 4 — 맵 배치 괴물

### 4-1. 정의

맵 배치 괴물은 새벽에 특정 지역에 생성되어 지도에 존재하는 지속 개체다.

랜덤 조우 괴물과 달리 처치되거나 이동 이벤트가 발생하기 전까지 맵에 남는다.

### 4-2. 저장 데이터

```yaml
instance_id:
monster_id:
location_id:
current_hp:
body_damage: []
status_effects: []
alert_state:
last_encounter_result:
escaped_by_party:
spawn_day:
```

### 4-3. 지속 규칙

- 피해와 상태이상은 다음 조우에도 유지한다.
- 도주해도 괴물은 처치되지 않는다.
- 재조우 시점과 확률은 괴물 AI, 장소 구조와 도주 결과에 따라 달라진다.
- 괴물은 특정 세계 이벤트나 이동 규칙으로 다른 지역에 갈 수 있다.
- 해당 지역 방문 시 조우 자체는 생략할 수 없다.

### 4-4. 도주 후 재조우

도주 성공은 조우를 끝내지만 지역에서 괴물을 제거하지 않는다.

재조우 후보:

- 지역 필수 이벤트 전
- 자유 탐색 중
- 심층 진입 시
- 귀환 중
- 다음 방문

세부 확률은 Combat·Monster AI 문서가 소유한다.

### 4-5. 필수 순서

장소별로 다음과 같은 순서를 사용할 수 있다.

```yaml
mandatory_sequence:
  - map_monster
  - region_trial
```

또는:

```yaml
mandatory_sequence:
  - region_trial
  - map_monster
```

둘 다 반드시 처리하지만 장소 연출에 따라 순서를 달리한다.

## SECTION 5 — 선택지 생성 규칙

### 5-1. 기본 구조

특수 선택지는 다음 요소의 조합으로 생성한다.

```text
핵심 스탯
+ 장비 기능 태그
+ 보조 스탯 또는 단서
+ 상태·환경 보정
```

### 5-2. 선택지 개방

선택지 개방은 다음 중 하나를 사용할 수 있다.

- 스탯 조건만 충족
- 장비 태그만 충족
- 스탯과 장비 태그 모두 충족
- 고유 단서 또는 플래그 충족

### 5-3. 판정과 결과 품질

```text
선택지 개방
→ 요구 조건 확인

성공 판정
→ 핵심 스탯 + 장비 보정 + 상태 보정

결과 품질
→ 성공도 + 보조 스탯 + 단서 + 사용 장비
```

한 선택지의 핵심 판정 스탯은 원칙적으로 하나만 사용한다.

### 5-4. 스탯 역할

| 스탯 | 탐험 역할 |
|---|---|
| 전투 | 강제 돌파, 위협 제압, 파괴 |
| 탐색 | 흔적, 자원, 숨은 공간, 매복 발견 |
| 기술 | 기계, 전자, 잠금장치, 시설 조작 |
| 지능 | 위험 분석, 단서 해석, 결과 품질 개선 |
| 제작 | 수리, 즉석 도구, 구조 보강 |
| 대화 | 세력·생존자 협상과 설득 |
| 간호 | 부상 진단, 의료품 사용, 치료 판정 |
| 매력 | 관계·유인·심리 선택지 일부 |

### 5-5. 예시

```yaml
choice:
  text: 잠긴 약품 보관함을 조용히 분해한다
  primary_stat: 기술
  difficulty: 어려움
  open_if:
    any_tag: [PRY, REPAIR, TOOLKIT]
  equipment_bonus:
    PRY: 1
    REPAIR: 1
    TOOLKIT: 2
  secondary_stat:
    stat: 지능
    effect: 성공 시 추가 의료품 후보 공개
```

장비가 없더라도 높은 기술로 시도할 수 있는 사건이 있을 수 있으나, 실패 위험이나 소음 대가가 커진다.

## SECTION 6 — 장비 기능 태그

### 6-1. 목적

아이템별 전용 선택지를 모든 이벤트에 수작업으로 작성하지 않기 위해 기능 태그를 사용한다.

### 6-2. 기본 태그 후보

```text
CUT
PRY
BREAK
REPAIR
TOOLKIT
LIGHT
OBSERVE
COMMUNICATE
TREAT
DISTRACT
RESTRAIN
PROTECT
CLIMB
SIGNAL
HACK
```

### 6-3. 태그 원칙

- 아이템은 하나 이상의 태그를 가질 수 있다.
- 이벤트는 허용 태그를 지정한다.
- 태그는 자동 성공이 아니라 선택지 개방 또는 보정을 제공한다.
- 같은 태그라도 아이템별 보정, 소음, 내구도 비용이 다를 수 있다.
- 합리적 창의성은 허용하되 이벤트가 지원하는 태그 범위 안에서 처리한다.

### 6-4. 예시

```yaml
item_id: ITEM_CROWBAR
function_tags: [PRY, BREAK, MELEE]
```

```yaml
item_id: ITEM_TOOLBOX
function_tags: [REPAIR, TOOLKIT, PRY]
```

## SECTION 7 — 기벽과 탐험

### 7-1. 일반 원칙

기벽은 탐험 선택지, 판정, 피로, 스트레스, 지역 진입과 귀환에 영향을 줄 수 있다.

기벽 자체의 분류, 효과와 제거 난이도는 각 Quirk DB가 소유한다.

### 7-2. 일반 기벽 분류

- 육체적 문제
  - 신체 외부
  - 신체 내부
- 정신적 문제
  - 정신이상
  - 욕구폭발·충동
  - 트라우마

### 7-3. 제거 방식 인터페이스

기벽은 일괄적인 단일 규칙으로 제거하지 않는다.

각 기벽은 다음 정보를 개별 정의해야 한다.

```yaml
removal_difficulty:
treatment_type:
required_item_category:
required_facility:
care_check:
failure_cost:
repeatable:
```

권장 난이도 계층:

- 쉬움: 간호 판정만으로 제거 가능
- 보통: 간호 판정 + 일반 의료품 또는 적절한 기호품
- 어려움: 높은 간호 판정 + 전문 의료품 또는 적합한 시설
- 전문적 치료: 전문 의료품 + 높은 간호 판정 + 의무실 또는 특수 설비
- 특수: 이벤트, 유물, 고유 능력 또는 장기 조건

### 7-4. 복잡도 제한

기벽 제거를 다단계 진행도로 관리하지 않는다.

원칙적으로 한 번의 치료 행동에서 판정과 자원 소비를 처리한다.

- 성공: 제거
- 부분 성공: 일시 완화 또는 부작용 감소
- 실패: 자원 일부 소모 또는 다음 재시도 조건

정확한 결과는 기벽별 데이터가 정의한다.

### 7-5. R18 기벽 확장

R18 기벽은 다음 계열을 사용할 수 있다.

- 육체
  - 신체개조
  - 감도개발
  - 내구성
- 정신
  - 성벽
  - 상식개변
  - 욕구

R18 기벽도 제거 가능 여부와 난이도를 개별 정의할 수 있어야 한다.

다만 공통 Quirk DB가 R18 문서에 의존하지 않으며, R18 치료 규칙은 R18 모듈이 소유한다.

현재 R18 Quirk DB의 `간호 제거 배제` 전제는 최신 설계와 충돌하므로 후속 개정 대상으로 둔다.

---

## SECTION 8 — 지역 자원과 반복 탐색 연결

세부 드롭 계산은 후속 `MECH_Loot_Distribution`이 소유한다.

탐험 시스템은 다음 정보만 전달한다.

- 장소
- 위험도
- 잔존 자원도
- 선택한 탐색 방식
- 탐색 성공도
- 집중 탐색 카테고리
- 장비·스탯 보정
- 심층 단계

잔존 자원도가 고갈 상태여도 기본 자원 획득은 가능하다.

풍부한 지역은 추가 보상 슬롯 또는 수량 보너스를 제공하고, 고갈 지역은 추가·고급 보상 효율이 낮아진다.

---

## SECTION 9 — R18 모듈 연결

공통 탐험 시스템은 R18 모듈 없이 완전하게 작동한다.

R18 ON에서는 다음 확장을 추가할 수 있다.

- R18 지역 이벤트 후보
- R18 전용 선택지
- R18 기벽에 따른 선택지 변화
- R18 Item Extension 태그
- 의복 게이지·굴복도·욕구 관련 전투 결과

R18 OFF에서는 해당 후보와 확장 효과를 제외하며 공통 탐험 흐름은 유지한다.
