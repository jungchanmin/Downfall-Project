# R18 전조·Bark·Narration 제작 정책 v1.0

## 0. 목적

전조는 수치 상태창이 아니라 전투 장면 안에서 전달한다.

```text
비권장:
[예상 공격: Contact / Chest / 신뢰도 높음]

권장:
“저 녀석, 갑자기 거리를 좁혀 오고 있어.”
괴물의 시선이 가슴께에 오래 머물렀다.
```

그러나 모든 기술 × 괴물 × 생존자 × 상태 조합마다 고유 Bark를 작성하지 않는다.

핵심 원칙은 다음과 같다.

```text
전조 아키타입은 재사용
Narration은 기본 전달 수단
Bark는 고가치 순간에만 사용
캐릭터 고유 Bark는 제한된 대표 상황만 소유
```

---

## 1. 전조 출력 계층

전조는 다음 네 계층으로 구성한다.

### 1-1. 물리적 Cue

괴물의 실제 움직임·시선·거리·준비 동작이다.

- 거리를 좁힘
- 한쪽 어깨를 내림
- 손을 들어 올림
- 목표 부위를 응시함
- 뒤로 돌아 들어감
- 체액·기관·도구를 준비함

기술마다 최소 1개의 물리적 Cue를 가진다.

### 1-2. 공용 Narration

전조 아키타입에 연결된 재사용 가능한 묘사다.

```text
괴물의 상체가 앞으로 기울었다.
손끝이 천천히 들려 올라왔다.
시선이 한곳에 고정되었다.
```

공용 Narration이 전조 전달의 기본값이다.

### 1-3. 상황 Bark

활성 생존자나 대기 생존자가 의미 있는 판단을 말로 전달한다.

```text
“거리를 좁혀오고 있어.”
“뒤를 잡으려는 거야!”
```

모든 합에 Bark가 발생하지 않는다.

### 1-4. 고유 Bark

특정 캐릭터의 직업·성격·관계가 드러나는 대표 대사다.

```text
탐정:
“손보다 어깨를 봐. 몸을 틀 준비를 하고 있어.”

펜싱 선수:
“중심이 앞에 쏠렸어. 들어오는 순간을 노려.”
```

고유 Bark는 제작비가 높은 콘텐츠로 분류한다.

---

## 2. 전조 아키타입

기술별 문장을 처음부터 작성하지 않고, 공격 접근 방식에 따라 아키타입을 공유한다.

초기 권장 아키타입:

| ID | 의미 |
|---|---|
| `TEL_CLOSE_DISTANCE` | 거리 접근 |
| `TEL_FRONTAL_GRAB` | 정면 붙잡기 |
| `TEL_REAR_APPROACH` | 후방 접근 |
| `TEL_PART_FIXATION` | 특정 부위 응시·노림 |
| `TEL_HAND_RESTRAINT` | 손·팔 점유 준비 |
| `TEL_FULL_RESTRAINT` | 전신 구속 준비 |
| `TEL_FLUID_PREP` | 체액 공격 준비 |
| `TEL_MUTATION_PREP` | 변이·신체 개조 준비 |
| `TEL_COGNITION_FOCUS` | 시선·음성·인지 간섭 준비 |
| `TEL_FINISHER_PREP` | 함락·결정타 준비 |

하나의 기술은 1~3개의 전조 아키타입을 조합할 수 있다.

```yaml
telegraph_profile:
  archetypes:
    - TEL_CLOSE_DISTANCE
    - TEL_FRONTAL_GRAB
    - TEL_PART_FIXATION
  target_part: Chest
  counter_tags:
    - FrontalGrab
    - ChestTarget
```

---

## 3. 제작 예산 등급

### Tier 0 — 시스템 필수

모든 전조 아키타입이 반드시 가진다.

- 공용 Narration 2~4개
- 물리적 Cue 태그
- 모호한 출력 1~2개
- 인지 교란용 왜곡 가능 여부

### Tier 1 — 괴물 정체성

괴물 종 또는 네임드 보스가 가진다.

- 시그니처 Narration 1~3개
- 시그니처 기술 전조 1~2개
- 반복 조우 시 인지 가능한 고유 동작

### Tier 2 — 캐릭터 직업 Bark

직업·경력 아키타입별로 작성한다.

- 전투가
- 탐색가
- 의료인
- 기술자
- 대화·관찰 특화

개별 캐릭터마다 처음부터 전부 작성하지 않고 직업 Bark를 공유한 뒤 말투를 조정한다.

### Tier 3 — 캐릭터 고유 Bark

다음 경우에만 작성한다.

- 시그니처 괴물과의 조우
- 개인 트라우마·고유 사건
- 대표 카운터 기술 성공
- 치명적 오판
- 관계가 중요한 지원 장면

일반 기술 전조에는 Tier 3를 요구하지 않는다.

---

## 4. 권장 출력 비율

초기 콘텐츠 예산 기준:

```text
Narration 중심 출력: 약 70%
공용·직업 Bark: 약 25%
캐릭터 고유 Bark: 약 5%
```

이 비율은 정확한 확률이 아니라 제작 우선순위 기준이다.

Bark가 없는 합도 정상이다. 물리적 Cue와 Narration만으로 충분히 전조를 전달할 수 있어야 한다.

---

## 5. Bark 발생 조건

Bark는 다음 상황 중 하나 이상에서만 우선 발생한다.

- 처음 보는 전조를 감지함
- 기존 기술과 다른 시그니처 전조
- 카운터 가능한 기술을 보유함
- 대기 생존자가 중요한 정보를 보정함
- 인지 교란으로 잘못된 확신이 발생함
- 함락·결정타 전조
- 주도권 연속 유지 위험
- 관계·캐릭터성이 드러나는 상황

일반 반복 전조에는 Narration을 우선한다.

---

## 6. 정보 정확도 표현

### 6-1. 일반 판정 부족

모호한 Bark 또는 Narration을 사용한다.

```text
“뭔가 하려는 것 같은데….”
괴물의 움직임이 빨라졌지만 의도를 읽기 어려웠다.
```

### 6-2. 경험·기록 축적

같은 전조를 학습할수록 구체적인 표현을 해금한다.

```text
초기:
“거리를 좁히고 있어.”

숙련:
“오른손이 올라왔어. 정면에서 붙잡으려는 거야.”
```

### 6-3. 인지 교란

구체적이지만 틀린 Bark를 사용한다.

```text
“정면 공격이야. 확실해.”
```

거짓 Bark는 인지 교란 effect가 직접 지정하며, 일반 실패에서 사용하지 않는다.

---

## 7. 조합형 문장 정책

모든 문장을 완성형으로 개별 작성하지 않고 일부는 슬롯 조합으로 생성한다.

```yaml
telegraph_line_template:
  subject: ["괴물", "놈", "그것"]
  motion: ["거리를 좁혔다", "몸을 낮췄다", "손을 들어 올렸다"]
  focus: ["가슴께를", "손목을", "등 뒤를"]
  intent_hint: ["노리는 듯했다", "붙잡으려는 것처럼 보였다"]
```

출력 예:

```text
괴물이 몸을 낮추며 손목을 노리는 듯했다.
```

조합형 문장은 다음 제한을 따른다.

- 한 문장에 슬롯 2~3개까지만 조합
- 같은 단어 반복 방지
- 캐릭터 고유 말투에는 자동 조합을 제한
- 중요한 시그니처 장면은 수동 작성

---

## 8. 중복·피로 방지

### 8-1. 최근 사용 기록

```yaml
telegraph_output_history:
  recent_line_ids: []
  recent_bark_speakers: []
  repeat_penalty:
```

최근 사용 문장과 화자는 선택 가중치를 낮춘다.

### 8-2. Bark 쿨다운

같은 화자의 일반 Bark에는 짧은 쿨다운을 둔다.

치명적 전조·함락 기술·고유 사건은 쿨다운을 무시할 수 있다.

### 8-3. 침묵 허용

모든 전조에 대사가 필요하지 않다.

```text
물리적 Cue
+
짧은 Narration
```

만으로 처리하는 것을 정상 출력으로 인정한다.

---

## 9. 제작 단위

신규 괴물이나 기술을 추가할 때 필요한 최소 전조 콘텐츠는 다음과 같다.

### 신규 일반 기술

- 기존 전조 아키타입 1~3개 연결
- 공용 Narration 재사용
- 필요 시 기술 고유 Narration 1개
- 카운터 태그 지정

### 신규 괴물

- 기존 아키타입 재사용
- 시그니처 동작 1~2개
- 괴물 고유 Narration 2~4개
- 전용 Bark는 선택 사항

### 신규 생존자

- 직업 Bark 세트 연결
- 고유 Bark 3~6개만 우선 작성
- 전체 전조 조합별 고유 Bark 작성 금지

## 11. 검수 체크리스트

- 상태창식 정보 문구에 의존하지 않는가
- Narration만으로도 전조 전달이 가능한가
- 기존 아키타입을 재사용할 수 없는 이유가 있는가
- 신규 Bark가 실제로 새로운 캐릭터성·정보를 제공하는가
- 일반 실패와 인지 교란의 표현이 구분되는가
- 같은 기술의 문장이 반복되어 정답 알림처럼 느껴지지 않는가
- 시그니처 기술만 고유 콘텐츠 비용을 지불하는가
