# R18 기술 소유·접근·회상 규칙 v1.0

## 0. 핵심 결정

굴복 페이즈 기술에 포켓몬식 고정 장착 슬롯을 사용하지 않는다.

```text
학습한 기술
→ 생존자가 계속 보유

전투 전 장착하지 않은 기술
→ 이유 없이 사용 불가 처리하지 않음
```

대신 전투 중에는 현재 상황에서 실제로 실행 가능한 기술과 즉시 떠오르는 기술을 우선 노출한다.

```text
보유 기술 전체
-
자세·부위·상태로 실행 불가능한 기술
-
현재 전조와 무관한 기술
=
우선 대응 목록
```

플레이어는 필요하면 전체 보유 기술을 펼쳐 볼 수 있으나, 불필요한 기술 목록이 기본 UI를 압도하지 않게 한다.

---

## 1. 공용 행동과 개별 기술

### 1-1. 시스템 공용 행동

모든 생존자는 기술 보유 여부와 관계없이 다음 최소 행동을 사용할 수 있다.

- 버티기
- 거리 벌리기
- 도움 요청
- 교대 요청
- 행동 포기

공용 행동은 Service·Tease·Domination·Absorption·Defiance 기술로 취급하지 않는다.

### 1-2. 개별 기술

실제 굴복 기술은 생존자별로 다르게 보유한다.

```yaml
survivor_submission_skills:
  survivor_id:
  known_skills: []
  proficiency_by_category: {}
  proficiency_by_skill: {}
  signature_skills: []
```

모든 생존자가 같은 대표 기술 5종을 자동 보유하지 않는다.

5대 커맨드 분류는 전술 역할과 UI 분류를 제공할 뿐, 기술 소유권을 의미하지 않는다.

---

## 2. 초기 기술 결정

초기 기술은 다음 요소를 조합해 결정한다.

1. 직업·경력
2. 능력 스탯
3. 신체 경험·훈련
4. 성격·행동 성향
5. 초기 배경 플래그
6. 시작 시점의 Memory Log와 기벽

초기 기술 권장 범위는 2~4개다.

- 최소 2개 이상의 전술 역할을 제공
- 한 생존자가 모든 분류를 고르게 보유하지 않음
- 공용 행동으로 완전 행동 불능을 방지

예시:

```yaml
survivor_id: Rachel
known_skills:
  - SB_Service_ClosePressure
  - SB_Tease_WristTurn
  - SB_Defiance_BreathControl
signature_skills:
  - SB_Tease_WristTurn
```

## 3. 기술 학습과 해금

기술은 다음 경로로 해금할 수 있다.

### 3-1. 반복 경험

특정 괴물 기술과 전조를 반복 관찰하거나 피격·카운터하면 대응 기술 후보가 열린다.

```text
FrontalGrab 반복 관찰
+
관련 부위·행동 경험
+
Tease 숙련
→ 정면 붙잡기 카운터 기술 후보
```

### 3-2. 훈련

- 동료 훈련
- 직업·전문성 기반 훈련
- 아지트 훈련 설비
- 사건 보상

### 3-3. 커맨드 숙련

같은 분류 기술을 반복 사용하면 분류 숙련이 상승하고 일반 기술을 해금한다.

### 3-4. Memory Log

관련 축·부위·행위 경험이 기술 해금 조건이나 성공 보정을 제공한다.

### 3-5. 기벽 Return

기벽은 일반 기술 자체보다 다음을 제공하는 것을 우선한다.

- 기존 기술 변종
- 새로운 조건부 effect
- 실패 비용 변화
- 자세·카운터 범위 변화
- 주도권·상태이상 추가 효과

---

## 4. 전투 중 기술 접근

### 4-1. 실행 가능성 필터

다음 조건을 먼저 검사한다.

- 현재 자세
- 목표 부위 접근
- 행동 부위 사용 가능 여부
- 점유 상태
- 상태이상
- 자원·쿨타임
- 대상 조건

실행 불가능한 기술은 회색 처리하고 사유를 표시한다.

### 4-2. 상황 적합성 우선 노출

실행 가능한 기술 중 현재 전조와 관계가 높은 기술을 먼저 보여준다.

```text
상대 전조: FrontalGrab + ChestTarget
→ 해당 태그를 카운터하는 Tease 기술 우선 노출
→ 자세 탈출 Defiance 기술 다음 노출
→ 무관한 Absorption 기술은 접힌 목록으로 이동
```

### 4-3. 회상 단계

기술 접근성은 다음 세 단계로 표현한다.

| 단계 | 의미 |
|---|---|
| Instinctive | 즉시 떠오르고 안정적으로 실행 가능 |
| Practiced | 상황에 맞으면 자연스럽게 선택 가능 |
| Known | 배웠지만 드물게 사용해 기본 목록 후순위 |

`Known` 기술도 사용 불가가 아니다. 전체 목록에서 선택할 수 있다.

### 4-4. 전투 집중

외출 전 특정 기술을 ‘장착’하지 않는다.

대신 훈련이나 준비로 하나의 기술 또는 커맨드 분류에 `focus`를 줄 수 있다.

```yaml
preparation_focus:
  category: Tease
  skill_id: SB_Tease_WristTurn
```

집중 효과 후보:

- 첫 사용 시 부담 감소
- 전조 연계 우선 노출
- 낮은 숙련 기술의 실패 비용 완화
- 관련 Bark의 정확도 상승

집중하지 않은 기술도 계속 사용할 수 있다.

---

## 5. 기술별 위력 구조

```text
최종 위력
=
기술 기본 위력
+ 기술 숙련
+ 분류 숙련
+ 관련 능력
+ Memory Log·부위 경험
+ 기벽 Return
+ 자세·전조 적합성
+ 상태 보정
- 피로·탈진·점유·부상
```

기술 기본 위력은 반드시 기술 데이터가 소유한다.

기벽·로그는 기술의 정체성을 대신하지 않고 보정과 변종을 제공한다.

---

## 6. UI 원칙

기본 UI는 다음 순서로 정렬한다.

1. 현재 전조를 직접 카운터할 수 있는 기술
2. 현재 게이트에서 탈출하거나 방어할 기술
3. 주도권 상태에 적합한 능동 기술
4. 기타 실행 가능 기술
5. 실행 불가능 기술

플레이어가 전체 목록을 펼치면 모든 보유 기술과 비활성 사유를 볼 수 있다.

```text
기술이 보이지 않음
≠ 기술을 잊음

기본 목록에 없음
→ 현재 상황과 연관성이 낮음
```
