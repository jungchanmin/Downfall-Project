# R18 기벽별 완화 행동 매핑

## 1. 공통 규칙

- 영구적 완화 내성은 사용하지 않는다.
- 생존자별 일일 완화 횟수와 설비별 일일 이용 횟수를 동시에 검사한다.
- 횟수는 새벽 날짜 변경 시 초기화하며 이월하지 않는다.
- Risk를 억제하면 직접 연결된 Return도 함께 억제한다.
- `none` 기벽은 안정화·격리만 가능하며 단계 하향이나 제거는 불가하다.
- 강제 완화는 생존 위기에서만 허용하며 관계 비용을 남긴다.

## 9. 일일 이용 처리

```text
완화 시도
→ 생존자 일일 한도 확인
→ 설비 일일 한도 확인
→ 동의·저항 판정
→ 행동 판정
→ 성공 여부와 관계없이 문서가 정한 사용량 차감
```

권장 기본 방향:

- 일반 완화 행동: 생존자 1회 + 설비 1회 소모
- 시설 없는 안정 상담: 생존자 한도만 소모
- 안전 격리: 완화 한도가 아니라 별도 통제 한도 사용
- 역행 사건: 일일 한도 비대상, 사건 자체가 사용 제한
- 실패 시에도 기본적으로 이용 횟수는 소모한다.

## 10. Return 정지 원칙

- 스탯 Risk 억제 시 해당 스탯 Risk에 직접 대응하는 Return만 정지한다.
- 관계·사건 선택지 Return은 완화 행동별로 별도 지정한다.
- `stabilization`은 원칙적으로 Return을 정지하지 않지만, 특정 자동 사건 보상은 중단할 수 있다.
- 억제 범위가 불명확하면 Risk와 Return을 모두 정지하는 보수적 처리를 사용한다.

## 11. 검토 필요 항목

- 생존자별 기본 일일 한도
- 설비별 기본 일일 한도
- 실패 시 이용 횟수를 항상 소모할지 여부
- 안전 격리의 별도 통제 한도
- 시설 없는 상담이 설비 제한을 우회하는지 여부
- 기벽별 세부 Risk·Return 정지 목록
