# 🌑 Downfall Project Context Manifest (v1.0)
*이 문서는 프로젝트의 핵심 아키텍처와 내러티브 지침을 요약한 스냅샷입니다.*

## 1. 프로젝트 개요 (Core Pillars)
- **장르**: 이벤트 중심 턴제 생존 TRPG / 심리 및 코즈믹 호러.
- **배경**: 고립된 '오뱅마을'에서의 29일간의 생존.
- **핵심 테마**: 이해 불가능한 공포, 내부의 갈등, 자원 고갈, 미지의 제물.

## 2. 시스템 규칙 (Core Mechanics)
### 🕒 시간 및 턴 시스템
- **5단계 페이즈**: 새벽(세계/괴물 이동) → 아침(배분/파견/갈등) → 점심(외출탐색/거점방어) → 저녁(복귀/호감도) → 밤(불침번/심리).
- **턴제 원칙**: 모든 행동은 시간을 소모하며, 플레이어의 선택이 완료되어야 다음 페이즈로 진행됨.

### 🏷️ 태그 명명 규칙 (Standard Tag Schema)
*엔진 파싱을 위해 아래 규격을 반드시 엄수할 것.*
1. `#Stat_[스탯명]_[연산자][수치]`: (예: `#Stat_불만도_>=8`)
2. `#Trait_[특성명]`: 성격, MBTI, 기벽, 상태 (예: `#Trait_ENTJ`, `#Trait_부상`)
3. `#Loc_[장소명]`: (예: `#Loc_아지트_거실`, `#Loc_SameAs_NPC_A`)
4. `#Env_[변수]_[연산자][수치]`: (예: `#Env_식량_==0`)
5. `#Flag_[ID]_[상태]`: 연계 및 이벤트 플래그 (예: `#Flag_EV001_True`)

## 🖋️ 내러티브 가이드 (Downfall Voice)
- **문체**: 헤밍웨이식 하드보일드. 짧고 건조하며 서늘한 문장.
- **원칙**: **Show, Don't Tell.** "무섭다"고 말하지 말고 "떨리는 손끝의 감각"을 묘사할 것.
- **페르소나**: NPC의 직업/성격을 전문 용어 나열이 아닌 '세상을 보는 관점'으로 투영할 것.
- **6모자 기법**: AI는 텍스트 생성 시 각 캐릭터의 6모자 데이터를 주입받아 사고 체계를 결정함.

## 🎒 자원 및 경제 (Economy)
- **핵심 자원**: 식량(허기), 부품(제작/수리), 금화(거래), 명성(세력/전체).
- **장비**: 일반 장비(내구도 있음, 거래 가능), 유물(내구도 무한, 귀속, 강력하지만 대가 따름).

## 🗺️ 지리 정보 요약 (Geography)
- **안전**: 아지트(생존자), 성당(중립).
- **위험**: 상점가(분쟁), 마을회관(자경단), 경찰서(악마조직), 철물점.
- **매우 위험/금지**: 유흥가, 주거단지, 선착장, 병원, 공장, 백화점, 연구소.
