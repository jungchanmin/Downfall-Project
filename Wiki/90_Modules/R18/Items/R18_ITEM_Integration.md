# R18 아이템 확장 연결 규칙

## 1. 목적

이 문서는 R18 모듈을 공통 아이템 체계에서 단순히 제외하지 않고, 어떤 아이템이 R18 전용인지, 공통 아이템에 어떤 R18 확장을 덧붙일 수 있는지, R18 토글이 꺼졌을 때 어떻게 동작하는지를 정의한다.

## 2. 최우선 원칙

- 공통 아이템 체계는 R18 모듈 없이 완전하게 작동해야 한다.
- R18 모듈은 공통 Item DB를 참조할 수 있지만 공통 Item DB가 R18 문서에 의존하면 안 된다.
- 동일한 물건을 공통 DB와 R18 DB에 중복 등록하지 않는다.
- R18 전용 효과는 공통 효과와 분리된 확장 데이터가 소유한다.
- R18 토글 OFF에서는 전용 항목을 숨김·대체·비활성화하되 저장 데이터가 깨지지 않아야 한다.

## 3. R18 아이템 유형

### 3.1 R18 전용 아이템

R18 모듈에서만 등장하고 공통 빌드에서는 필요하지 않은 아이템.

예시 역할:

- R18 관계·욕구 이벤트용 기호품
- R18 전용 상태·기벽·기억 로그에 연결되는 소모품
- 굴복 페이즈 전용 보조 아이템
- R18 전용 의복 또는 장신구
- 성인 전용 장소·세력에서만 거래되는 물품

이 항목은 후속 `R18_ITEM_DB`가 소유한다.

### 3.2 공통 아이템의 R18 확장

공통 아이템은 그대로 유지하고, R18 모듈이 켜졌을 때만 추가 효과나 태그를 부여하는 방식.

예:

- 일반 의복에 `clothing_gauge_profile` 추가
- 보호 장비에 의복 손상 저항 또는 노출 변화 규칙 추가
- 의료품에 R18 전용 상태이상 치료 태그 추가
- 기호품에 R18 관계 이벤트 선택지 추가
- 유틸리티 장비에 특정 R18 이벤트 조사 선택지 추가

이 항목은 후속 `R18_ITEM_EXTENSION_DB`가 소유한다.

### 3.3 공통 효과와 R18 효과를 모두 가진 아이템

한 아이템이 공통 기능과 R18 기능을 동시에 가질 수 있다.

예:

- 공통: 스트레스 완화 기호품
- R18 ON: 특정 관계 이벤트 또는 욕구 관련 선택지 해금

이 경우 아이템 ID는 공통 Item DB에 하나만 존재하고, R18 확장 문서가 같은 ID를 참조한다.

## 4. 성인잡지의 위치

`성인잡지`는 일반 장비나 전투 무기가 아니다.

권장 분류:

```text
상위: 사용 아이템
하위: R18 전용 기호품
주 사용처: 관계 이벤트 / 욕구 관련 이벤트 / 거래 / 특정 대화 선택지
소모 방식: 이벤트별 소비 또는 사용 횟수
```

가능한 기능 방향:

- 특정 NPC의 스트레스 또는 욕구 관련 이벤트에 사용
- R18 관계 이벤트의 선택지·대화 해금
- 유흥가·암시장 거래품
- 일부 NPC에게는 긍정, 일부 NPC에게는 불만·거부·관계 악화 가능

사용 즉시 단순 수치만 회복하는 만능 아이템으로 만들지 않는다. NPC 성격·관계·상황에 따라 반응이 달라져야 한다.

R18 토글 OFF에서는 다음 중 하나를 사용한다.

- 획득 테이블에서 제외
- 동일 가치의 일반 기호품으로 대체
- 이미 보유 중이면 비활성화 상태로 보존

## 5. R18 전투 장비와 의복

### 5.1 공통 무기·방어구

공통 무기와 방어구는 `ITEM_Equipment_DB`에만 등록한다.

R18 전투에서 추가로 필요한 데이터만 확장한다.

```yaml
item_id:
r18_enabled: true
clothing_gauge_profile:
clothing_damage_resistance:
sensitivity_modifier:
submit_skill_modifier:
r18_status_resistance:
r18_tags: []
```

### 5.2 의복 게이지 연결

R18 Combat의 의복 게이지는 아이템 자체의 일반 내구도와 동일하지 않다.

- 일반 내구도: 아이템 파손·수리 가능 여부
- 의복 게이지: 해당 전투에서의 R18 연출·성감 증폭 상태

전투 종료 시 의복 게이지는 초기화할 수 있지만, 사건 결과가 실제 장비 손상으로 확정된 경우에만 일반 내구도에 반영한다.

이를 분리하지 않으면 R18 피격이 매번 공통 장비를 영구 파괴하는 문제가 발생한다.

### 5.3 R18 전용 전투 보조품

후속 설계에서 다음 유형을 둘 수 있다.

- R18 상태 저항 또는 완화
- 의복 게이지 변화 억제
- 특정 굴복 전용 기술 강화
- 특정 R18 기벽 또는 기억 로그와 상호작용
- 포획·속박·최면 등 R18 상태 대응

단, 장비 하나가 R18 전투를 무력화하거나 굴복도·흥분 게이지를 완전히 차단해서는 안 된다.

## 7. 토글 규칙

### R18 ON

- 공통 Item DB 로드
- R18 전용 Item DB 추가 로드
- 공통 아이템의 R18 확장 데이터 병합
- R18 전투·관계·이벤트 태그 활성화

### R18 OFF

- 공통 Item DB만 사용
- R18 전용 아이템은 드롭·상점·이벤트 후보에서 제외
- 공통 아이템의 R18 확장 필드는 무시
- R18 전용 아이템 보유 세이브는 삭제하지 않고 비활성화 또는 대체 처리

## 8. 금지 규칙

- 공통 Item DB에 노골적 R18 효과를 직접 기록하지 않는다.
- 동일 아이템을 공통 DB와 R18 DB에 복제하지 않는다.
- R18 OFF에서 공통 전투·인벤토리·세이브 로드가 실패하면 안 된다.
- R18 장비가 공통 장비보다 무조건 상위 성능이 되면 안 된다.
- 의복 게이지와 일반 내구도를 동일 변수로 사용하지 않는다.

## 9. 미확정 항목

- R18 전용 아이템 실제 목록
- R18 의복 분류와 장착 슬롯
- 의복 게이지 프로파일 규격
- R18 전용 전투 보조품 효과
- 성인잡지의 구체 수치와 NPC별 반응
- R18 OFF 세이브 대체 정책의 구현 방식

## 10. 후속 순서

1. 공통 `ITEM_Equipment_DB`의 1차 후보 확정
2. 공통 아이템 중 R18 확장이 필요한 항목 식별
3. R18 전용 관계·기호품 후보 검토
4. R18 전투 보조품과 의복 확장 필드 정의
5. R18 ON/OFF 세이브 호환 테스트
