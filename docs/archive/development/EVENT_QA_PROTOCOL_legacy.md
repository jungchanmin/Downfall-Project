# 🔬 이벤트 품질 검수 프로토콜 (v1.0)

> **목적**: AI 컨베이어 벨트가 출고한 이벤트가 ① **다운폴 톤(셜리 잭슨)을 유지**하면서 ② **DB로 관리·로드되는 개발 산출물로 정상 동작**하는지를 이중으로 검증.
>
> **원칙**: 검수는 *주관적 인상* 이 아니라 *체크리스트 통과 여부* 로 결정한다. 통과 못 한 항목은 자동으로 "재작성 큐" 로 회송.

---

## 0. 검수 파이프라인 (전체 흐름)

```
[AI 출고]
   ↓
[Gate A] 자동 정적 검사 (스크립트)         ← 기계가 한다
   ↓ 통과
[Gate B] 내러티브 검수 (휴먼/AI 교차)      ← 디렉터가 한다
   ↓ 통과
[Gate C] 시스템 정합성 검수 (DB 시뮬)      ← 엔지니어가 한다
   ↓ 통과
[Gate D] 인게임 플레이테스트 (구현 후)
   ↓
[승인 → 05_Events/ 본 DB 등재]
```

각 게이트는 **블로킹** 이다. 한 게이트 미통과 시 이후 게이트로 보내지 않는다.

---

## 1. Gate A — 자동 정적 검사 (Static Lint)

스크립트(파이썬 등)로 자동화 가능. **사람의 감각을 낭비하지 않는다**.

### A-1. 스키마 준수
- [ ] 이벤트 ID 가 `EVT_[Track][번호]_[해시]` 패턴인가
- [ ] Track A/B 헤더 필드가 모두 채워졌는가 (빈칸·`TODO`·`...` 검출 시 즉시 회송)
- [ ] 시네마틱 8단계 슬롯이 모두 차 있는가

### A-2. 태그 명명 규칙
`ConText_Manifest.md` 의 5가지 스키마와 정확히 일치해야 함.
- [ ] `#Stat_*_[연산자][수치]` — 알려진 스탯명 화이트리스트에만 존재
- [ ] `#Trait_*` — `Mental_Ailments.md` 에 존재하는 ID와 매칭
- [ ] `#Loc_*` — `Oebeng_Village_Lore.md` 의 15구역 화이트리스트
- [ ] `#Rel_*` — 해당 캐릭터 로어 바이블의 관계 항목에 존재
- [ ] `#Flag_Memory_*` — 명명이 `Flag_Memory_[행위자]_[사건]` 형식

### A-3. 수치 새너티
- [ ] 단일 이벤트의 스탯 변동량이 `Resource_Logic.md` 가 정의한 1회 상한선 이내인가 (예: 식량 ±3 초과 금지)
- [ ] 트리거 조건이 *현재 게임 상태에서 평가 가능* 한 변수만 참조하는가 (정의되지 않은 변수 색출)

**자동 검사 미통과 시**: 회송 사유를 `lint_errors.json` 으로 출력하여 AI에게 다시 입력. 이 단계에서 사람이 개입할 필요 없음.

## 2. Gate B — 내러티브 검수 (Tone & Voice)

**여기서 다운폴이 다운폴인지가 결정된다.**

### B-1. 셜리 잭슨 4지침 발현 (각 항목 1~5점)

| 지침 | 5점 (탁월) | 3점 (충족) | 1점 (실패) |
|---|---|---|---|
| 🏠 공간의 악의 | 공간이 인물의 약점을 정확히 조롱함 | 공간 묘사가 분위기적으로 압박함 | 단순 배경 묘사 |
| 🧠 신경증적 투사 | 인물의 기벽이 환경 해석을 왜곡 | 인물의 심리가 묘사에 반영됨 | 외부 관찰자 톤 |
| 🚪 일상의 배신 | 통제되던 사물이 미세하게 어긋남 | 분위기 변화가 있음 | 노골적 괴물·피 등장 |
| 🗣️ 무력한 목소리 | 대사가 허공으로 흩어져 고립감 강조 | 대사가 상황을 진전시키지 못함 | 대사가 상황 해결책을 제시 |

**합격선**: 4지침 평균 3.5점 이상, 어느 지침도 2점 이하 없음.

### B-2. 캐릭터 보이스 일치
- [ ] 등장 인물의 대사·내심이 해당 캐릭터의 로어 바이블 `Declared` / `Actual` 간극을 활용했는가
- [ ] 인물의 기벽(`#Trait_*`)이 행동/지각에 반영되었는가 (단순 라벨 부착이 아니라)
- [ ] 캐릭터의 `Primary Setting` 과 충돌·공명하는가

### B-3. 클리셰 회피
- [ ] "거울 속에 누가 있다", "어둠 속에 눈동자", "갑자기 문이 쾅" 같은 진부 호러 트로프를 피했는가
- [ ] 노골적 폭력 묘사 없이 공포가 작동하는가
- [ ] 인물이 "그것은 형언할 수 없었다" 같은 도피적 서술로 책임을 회피하지 않는가

### B-4. AI 교차 검증 (Self-Consistency)
다운폴 톤에 대해 **무지한 별도 AI 세션** 에 다음을 묻는다:
> "이 텍스트의 화자는 어떤 작가의 영향을 받은 것 같습니까? 분위기 키워드 5개를 뽑아주세요."

- 합격: 키워드 중 ≥3개가 ['건조한', '심리적', '일상적', '서늘한', '의인화', '강박', '미세한 어긋남'] 류와 의미 일치
- 불합격: '잔혹한', '직설적', '액션', '해피' 같은 키워드 등장

### B-5. 출력
- 각 항목 점수와 **회송 사유 코멘트** 를 `qa_narrative_report.md` 로 정리
- 점수 하한 미달 시 → **재작성 큐** 로 보내며, 어느 지침이 약했는지 컨베이어 벨트에게 명시적으로 피드백

## 3. Gate C — 시스템 정합성 검수 (DB Sanity)

**여기서 "예쁜 글이지만 코드로 안 돌아감" 이 걸러진다.**

### C-1. DB 적재 가능성
이벤트가 다음 형태의 JSON으로 *손실 없이 직렬화* 되는가:

```json
{
  "id": "EVT_B007_a1f3",
  "track": "B",
  "trigger": {
    "phase": "점심",
    "location": "#Loc_상점가",
    "actor": "Gavin",
    "conditions": ["#Stat_스트레스_>=5", "#Trait_통제강박"]
  },
  "narrative": { "title": "...", "body": "...", "stages": [...] },
  "choices": [
    {
      "id": "ch1",
      "label": "...",
      "cost": {...},
      "resolution": {
        "success": { "stat_delta": {...}, "flags_emit": [...] },
        "fail":    { "stat_delta": {...}, "flags_emit": [...] }
      }
    }
  ],
  "memory_emit": ["#Flag_Memory_Gavin_..."]
}
```
- [ ] 모든 효과가 *enum화 가능한 키* 로 표현되는가 (자유 서술 효과 금지)
- [ ] 한 이벤트 한 페이로드 — 다른 이벤트 본문 안에 본문이 끼어들지 않음

### C-2. 그래프 정합성
- [ ] 발행하는 `#Flag_Memory_*` 가 어딘가에서 **소비** 되는가 (orphan flag 색출)
- [ ] 트리거 조건의 `#Stat_*_>=N` 의 N이 게임 내 도달 가능한 값인가 (영원히 트리거되지 않는 데드 이벤트 색출)
- [ ] 이벤트 결과가 다른 이벤트의 트리거를 만족시키며 *체인* 을 형성할 수 있는가 (고립 이벤트 비율 < 30%)

### C-3. 밸런스 분포
파일럿 N개를 모은 뒤:
- [ ] 스트레스 +/- 의 일평균 합산이 임계점 도달 시점과 정합한가
- [ ] 자원 +/- 의 일평균이 결핍/과잉 어느 한쪽으로 편향되지 않았는가
- [ ] 각 캐릭터의 호감도 변동 합이 한쪽으로 쏠리지 않았는가

### C-4. 회귀 테스트
- [ ] 기존 이벤트와 **충돌하는 효과** 를 가진 신규 이벤트인가 (예: 같은 페이즈에 같은 캐릭터를 두 곳에 배치하라고 하는 모순)

## 4. Gate D — 플레이테스트 (구현 이후)

게임이 실제로 빌드된 이후 단계. 본 문서는 빌드 전 게이트(A·B·C)에 집중하되, D를 위한 **메타데이터** 만 미리 부착해 둔다.

각 이벤트에 다음 라벨 부착:
- `playtest_priority`: 1~3 (테스터가 우선 확인할지)
- `expected_emotion`: ['불안', '안도', '죄책감', '오만한 확신' 등 1~2개]
- `failure_mode_to_watch`: "이 이벤트가 무너지면 어떻게 무너지는지" 한 줄

테스트 시 **실제 플레이어 감정 ↔ expected_emotion 일치율** 을 측정. 일치율 < 60% 면 내러티브 재작성.

## 5. 검수 운영 모델

### 5-1. 역할 분담 (1인 개발 시 시간 분리)
| 역할 | 무엇을 보는가 | 시간대 |
|---|---|---|
| **Lint Bot** (스크립트) | Gate A | 출고 즉시, 자동 |
| **디렉터(나)** | Gate B | 정해진 검수 슬롯에 묶어서 — 한 번에 5~10개씩 |
| **엔지니어(나)** | Gate C | 빌드 패스에 묶어서 — 주 1회 일괄 |
| **테스터(나/외부)** | Gate D | 마일스톤 빌드마다 |

→ **롤을 시간으로 분리하는 것이 핵심.** 같은 사람이라도, 작가 모드와 엔지니어 모드를 한 번에 돌리면 둘 다 흐려진다.

### 5-2. 회송 처리
회송된 이벤트는 컨베이어 벨트의 Step 4(지침 점검)로 다시 들어간다. 단,
- **회송 사유가 명시적 피드백 문장** 이어야 함 ("톤이 약함" ❌ / "공간의 악의 1점: 폐가가 단순히 어둡다고만 묘사됨, 인물의 강박을 조롱하는 디테일 없음" ⭕)
- 동일 이벤트 3회 회송 시 → **폐기**, 시드 키워드부터 다시.

### 5-3. 합격 이벤트의 보관
승인된 이벤트는 `05_Events/` 산하에:
```
05_Events/
  by_day/Day_03/EVT_B007_a1f3.md       (사람이 읽는 본문)
  by_day/Day_03/EVT_B007_a1f3.json     (게임이 읽는 직렬화본)
  by_track/Track_B/...                  (심볼릭 링크)
  by_actor/Gavin/...                    (심볼릭 링크)
```

이중 저장(.md + .json)은 의도된 것: **사람과 기계가 같은 진실을 본다**.

## 6. 검수 메트릭 (Sprint 단위 계측)

매 스프린트마다 측정:
- **Gate A 1회 통과율** — 목표 ≥ 80%. 낮으면 컨베이어 벨트 프롬프트 보강.
- **Gate B 평균 셜리 잭슨 점수** — 목표 ≥ 3.7 / 5. 낮으면 라이팅 지침 강화.
- **Gate C 그래프 고립률** — 목표 ≤ 30%. 높으면 이벤트 간 연결 시드 추가.
- **재작성 평균 횟수** — 목표 ≤ 1.5회. 높으면 Step 2 키워드 단계 재설계.

---

## 7. 자동화 우선순위 (구현 시)

`tools/qa/` 산하에 다음 스크립트를 우선 구축:
1. `lint_event.py` — Gate A 전체. 입력: .md, 출력: errors.json
2. `extract_payload.py` — .md → .json 변환기. C-1 검사 동시 수행.
3. `flag_graph.py` — 모든 이벤트 스캔, orphan flag / dead trigger / 체인 시각화.
4. `tone_eval.py` — Gate B-4의 별도 AI 세션 자동 호출 래퍼.

**1과 2가 가장 ROI가 높다.** 먼저 이것들부터 만든다.

---

## 8. 합격선 요약 카드

> **이벤트가 본 DB에 등재되려면**, 다음을 모두 만족해야 한다:
> 1. Gate A 자동 검사 무결점 통과
> 2. Gate B 셜리 잭슨 평균 ≥ 3.5, 항목별 최저 ≥ 2점
> 3. Gate B 캐릭터 보이스 / 클리셰 / AI 교차검증 모두 통과
> 4. Gate C JSON 직렬화 무손실, orphan flag 0개
> 5. 회송 횟수 ≤ 3회

이 합격선 미달 이벤트는 *예쁘게 거절한다*. 다운폴은 양보다 톤이다.
