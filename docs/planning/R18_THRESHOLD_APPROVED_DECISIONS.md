# R18 Threshold Approved Decisions

## 2026-07-14 Director Decision

### 채택 구조

R18 메모리 로그와 기벽 획득에는 혼합형 임계값 구조를 사용한다.

```text
축 단계
→ 공통 Record / Proficiency / Milestone 임계값

개별 기벽 획득
→ required_axis_stage
→ required_part_level
→ required_event / required_flags

기벽 고착
→ 재획득 기록
→ 관련 축 단계
→ 기벽별 사건·특수 조건
```

### 공통 축 단계의 역할

- 전체 변질 진행도를 플레이어가 이해할 수 있는 공통 언어로 제공한다.
- 축마다 별도 규칙을 외워야 하는 부담을 줄인다.
- 실제 숫자는 전투·이벤트 발생 빈도 테스트 후 확정한다.
- 초기 `3 / 5 / 8` 예시는 테스트 후보이며 정본 수치가 아니다.

### 개별 기벽 조건의 역할

- 동일 축 내부에서 기벽의 희소성·심각도·발생 맥락을 구분한다.
- 관련 부위나 사건 없이 무관한 기벽이 발생하는 것을 방지한다.
- 강한 영구 기벽은 높은 단계·부위 기록·고유 사건을 추가 요구할 수 있다.
- 로그 단계 도달만으로 관련 기벽이 한꺼번에 부여되지 않는다.

### 고착 원칙

- 재획득 횟수만으로 자동 영구화하지 않는다.
- `temporary → persistent → permanent` 전이에는 관련 축 단계와 기벽별 조건이 함께 필요하다.
- `irreversible` 기벽은 획득 사건 자체가 영구 고착 조건을 포함할 수 있다.

### 괴물 성애 정합성

- 도주·기습 선택지 완전 하드락을 사용하지 않는다.
- 정신력 또는 자제 판정으로 성공·부분 성공·실패를 처리한다.
- 실패하면 정면 대치로 전환하되, 한 NPC가 파티 전체 선택을 자동 봉쇄하지 않는다.
- 괴물 조우 스트레스 면제 Return은 유지한다.
