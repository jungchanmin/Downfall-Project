# Item Equipment Candidate Draft

## 1. 목적

이 문서는 `ITEM_Equipment_DB.md` 작성 전에 일반 장비·사용 아이템 후보의 역할을 정리하는 검토 초안이다.

세부 수치, 가격, 드롭률, 내구도 최대값과 사용 횟수는 게임 테스트와 밸런스 패치 단계에서 확정한다.

## 2. R18 연결 코드

| 코드 | 의미 |
|---|---|
| `none` | R18 확장 불필요 |
| `optional` | 공통 기능은 완전하며 R18 ON에서 추가 이벤트·효과 가능 |
| `required` | 공통 아이템이지만 R18 전투·의복 시스템 연결 데이터가 필요 |
| `r18_only` | R18 모듈에서만 등록되는 전용 아이템 |

공통 Item DB에는 `none / optional / required` 판정만 기록하며, 실제 R18 효과는 `R18_ITEM_EXTENSION_DB`가 소유한다.

## 3. 설계 공통 원칙

- 한 아이템이 지나치게 많은 상황의 정답이 되지 않도록 한다.
- 유틸리티 장비는 범용 보정과 일부 특별 행동을 제공하되 자동 성공을 만들지 않는다.
- 전술품은 대표 용도를 제시하되 합리적인 창의적 사용을 허용한다.
- 의료품은 기벽·관계·영구 정신 손상을 손쉽게 제거하지 않는다.
- 무기와 보호 장비는 세부 수치보다 전투 역할과 위험 대응 범위를 우선 정의한다.
- R18 전용 성능이 공통 성능의 상위 호환이 되지 않도록 한다.

## 12. R18 분기 요약

### 12.1 R18 전용

현재 일반 장비 초안에서 확정적으로 R18 전용으로 분리한 항목은 `성인잡지`뿐이다.

이는 목록 확정이 아니라 구조 검증용 대표 사례다. 후속 R18 아이템 검토에서 관계 기호품, 전투 보조품, 전용 의복·장신구 후보를 별도로 다룬다.

### 12.2 R18 확장 필수

다음은 공통 보호 기능을 유지하되, R18 ON에서 의복 게이지·손상 프로파일 연결이 필요한 후보다.

- 두꺼운 외투
- 방검복
- 방탄조끼
- 화학 보호복

### 12.3 R18 확장 선택

대다수 일반 아이템은 R18 사건에서 추가 선택지를 제공할 수 있지만, 공통 기능만으로 완전하게 작동한다.

`optional` 판정은 실제 확장 효과를 반드시 넣는다는 의미가 아니라, 후속 검토 대상이라는 뜻이다.

## 13. 다음 승인 대상

디렉터 검토에서는 세부 수치보다 다음만 확인한다.

1. 후보 자체가 Downfall에 필요한가
2. 분류가 직관적인가
3. 역할이 다른 항목과 중복되지 않는가
4. R18 연결 수준이 과도하거나 부족하지 않은가
5. 별도 DB·Entity·Event로 이동해야 하는가

승인 후 `ITEM_Equipment_DB.md`에는 우선 상태 `candidate`로 등록하고, 실제 밸런스 수치는 테스트 단계에서 조정한다.
